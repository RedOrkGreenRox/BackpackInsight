# Архитектура "BackpackInsight Ecosystem"

Этот проект использует биологическую метафору для структуры приложения.
Мы отказались от стандартных названий (src, components, router) в пользу более органичных.

## Словарь терминов (Glossary)

| Наше название | Стандарт (SPA) | Описание |
| :--- | :--- | :--- |
| **Ground** | `src` | "Почва". Корневая папка исходного кода. Всё растет отсюда. |
| **Kernel** | `core` | "Ядро". Системный код, движок, который заставляет всё работать. |
| **Gen** | `Router` | "Геном". Отвечает за состояние организма. Определяет, какая Ветка (Branch) сейчас активна (экспрессируется) на основе URL. |
| **Seed** | `Component Interface` | "Семя". Интерфейс/Контракт. То, из чего может вырасти Ветка. Описывает методы `grow`, `root`, `uproot`. |
| **Roots** | `shared/layout` | "Корни". Базовые, неизменные части: глобальные стили, шрифты, Layout (сайдбар, фон), утилиты. |
| **Branches** | `pages/views` | "Ветви". Конкретные страницы (Профиль, Главная). Они растут из Корней и управляются Геном. |

## Жизненный цикл Ветки (Branch Lifecycle)

1. **Grow** (`render`): Ветка формирует свою визуальную структуру (HTML).
2. **Root** (`mount`): Ветка укореняется в DOM. Подключаются слушатели событий, запускаются соки (скрипты).
3. **Uproot** (`unmount`): Ветка удаляется. Слушатели снимаются, память очищается.
