# [Конфигурация pytest (pytest.ini)](../pytest.ini)

## Назначение
Файл `pytest.ini` — это центральный конфигурационный файл для тестового фреймворка **pytest**. Он задает правила автоматического обнаружения тестов и форматирования результатов для всего проекта BackpackInsight.

---

## Детальные настройки

### 1. Обнаружение тестов (Test Discovery)
*   **`testpaths = tests`**: Указывает фреймворку искать тесты в корневой директории `tests/`.
*   **`python_files = test_*.py`**: Искать файлы, начинающиеся с `test_`.
*   **`python_classes = Test*`**: Распознавать классы тестов.
*   **`python_functions = test_*`**: Распознавать функции тестов.

### 2. Параметры запуска (`addopts`)
*   **`-ra`**: Показывать краткий отчет по всем тестам, кроме пройденных (failed, skipped, xfailed).
*   **`--tb=short`**: Использовать сокращенный формат вывода ошибок (traceback), что удобно при большом количестве тестов.

### 3. Фильтрация предупреждений (`filterwarnings`)
*   **`ignore::DeprecationWarning`**: Игнорировать предупреждения об устаревании функций (часто возникают во внешних библиотеках, таких как SQLAlchemy или FastAPI), чтобы они не засоряли лог тестирования.

---

## Связи (Dependencies)
*   **Тесты**: Конфигурирует выполнение всех скриптов в папке [tests/](tests/test_api.md).
*   **Автоматизация**: Используется скриптом [run_tests.py](scripts/run_tests.md) для запуска полной проверки проекта одной командой.

## AI-контекст
*   Это **основной** конфиг. Хотя в подпапках (например, в `Backend/PlayerData/`) могут находиться свои `pytest.ini`, именно этот файл задает тон для глобального CI/CD процесса.
*   Если новый файл с тестами не подхватывается pytest, проверьте, соответствует ли его имя шаблону `test_*.py`.
