# [Запуск Docker-окружения (run_docker.py)](../../scripts/run_docker.py)

## Назначение
Сервисный скрипт для управления жизненным циклом Docker-контейнеров проекта. Он автоматизирует процесс сборки, запуска и проверки готовности (health-check) всех сервисов.

---

## Ключевая логика

### 1. Режимы работы
*   **Локальный**: Использует `docker-compose.yml` с пробросом портов и монтированием исходников для Hot Reload.
*   **Серверный**: Использует `docker-compose.server.yml` с лимитами ресурсов и закрытыми портами БД.

### 2. Режим "Паранойи" (`PARANOID_MODE`)
При включении этого режима скрипт перед запуском:
1.  Принудительно удаляет старые контейнеры.
2.  Выполняет сборку с параметром `--build`.
3.  Использует `CACHE_BUST` (таймстамп) для обхода кэша Docker при необходимости.

### 3. Умные Health-checks
Скрипт не просто ждет запуска процесса, а выполняет HTTP-запросы к эндпоинтам:
*   **API**: Ожидание до 180 секунд. Большой таймаут обусловлен необходимостью выполнения миграций и синхронизации ~1100 предметов при первом старте.
*   **Web**: Ожидание до 60 секунд.

---

## AI-контекст
*   Скрипт включает "UTF-8 enforcement" для Windows, что предотвращает появление кракозябр в консоли при выводе системных ошибок.
*   При падении health-check скрипт автоматически выводит хвост логов (tail) бэкенда для быстрой диагностики.
