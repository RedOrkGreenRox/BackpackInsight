# [Генератор карты структуры (generate_structure.py)](../../scripts/generate_structure.py)

## Назначение
Генерирует [Центральный Хаб структуры (structure.md)](../structure.md) — дерево исходников проекта, где каждый задокументированный файл становится ссылкой на свой `.md`. Реализует принцип навигации сетевой документации из [README](../../README.md).

## Подробное описание
*   `IGNORE_DIRS`/`IGNORE_FILES` — что пропускать (ассеты, кэши, миграции, сам `structure.md` и т.п.).
*   `BEAUTIFUL_NAMES` — карта «файл → человекочитаемое имя» для узлов дерева.
*   `find_doc_file(relative_path)` — **ключевая функция зеркала 1:1**: по пути исходника вычисляет ожидаемый путь дока (`Frontend/Web/...` → `docs/frontend/...`, нижний регистр папок, basename+`.md`). Именно она определяет правило размещения доков.
*   `generate_list_tree(...)` — рекурсивно строит дерево, проставляя ссылки на найденные доки или «голый» текст-заглушку для недокументированных.
*   `main()` — собирает шапку (ссылки на хабы бэкенда/фронтенда), дерево, примечание и подпись; пишет в `docs/structure.md`.

## Связи (Dependencies)
*   Парный инструмент контроля качества — [линтер документации (check_docs.py)](check_docs.md).

## AI-контекст
*   **Это инструмент-владелец карты**: после любых изменений в дереве исходников или добавления доков запускайте `python3 scripts/generate_structure.py`. Правило зеркала из `find_doc_file` — канон размещения `.md`; новые доки кладите туда, куда ожидает эта функция, иначе они отобразятся в карте как заглушки.
