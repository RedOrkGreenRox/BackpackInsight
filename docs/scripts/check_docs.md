# [Линтер документации (check_docs.py)](../../scripts/check_docs.py)

## Назначение
Скрипт `check_docs.py` — это инструмент автоматического контроля качества «сетевой документации» проекта. Он гарантирует, что документация остается актуальной, связной и соответствует реальному коду.

---

## Проверки (Checks)

Скрипт выполняет 6 типов объективных проверок:

### 1. `[LINKS]` — Битые ссылки
Проверяет все внутренние ссылки во всех `.md` файлах. Если ссылка ведет на несуществующий файл или якорь, проверка не проходит.

### 2. `[TREE]` — Достижимость (Орфаны)
Строит граф ссылок, начиная от `README.md` и `structure.md`. Выявляет «файлы-сироты» — документы, на которые нет ни одной ссылки из центральных узлов.

### 3. `[MIRROR]` — Зеркальность (1:1)
Гарантирует соблюдение принципа проекта: у каждого файла исходного кода (`.ts`, `.py`, `.scss`) должен быть соответствующий файл документации.

### 4. `[STRUCT]` — Валидация структуры
Проверяет, что каждый файл документации содержит ссылку на свой исходник в первом заголовке (H1) и имеет ключевые секции (Назначение, Связи).

### 5. `[COMPLETE]` — Покрытие символов
Анализирует исходный код и проверяет, упомянуты ли функции, классы и CSS-селекторы в тексте документации. Порог прохождения — **50%**.

### 6. `[TRUTH]` — Истинность (Детектор лжи)
Самая важная проверка. Ищет в документации упоминания методов или селекторов, которых **нет** в коде. Это позволяет отлавливать устаревшие описания после рефакторинга.
