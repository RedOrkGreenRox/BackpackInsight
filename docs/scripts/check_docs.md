# [Линтер документации (check_docs.py)](../../scripts/check_docs.py)

## Назначение
Автоматическая объективная проверка всей сети сетевой документации. Заменяет ручной перебор: один прогон валидирует целостность, связность, покрытие и истинность.

## Что проверяет
*   **[LINKS]** — битые ссылки во всех `.md` (включая [README](../../README.md)).
*   **[TREE]** — достижимость каждого дока обходом (BFS) от README/[structure](../structure.md); ловит «осиротевшие» узлы.
*   **[MIRROR]** — у каждого исходника (`.py/.ts/.scss/.js`, кроме `__init__.py`) есть `.md`.
*   **[COMPLETE]** — покрытие символов: % функций/классов/CSS-селекторов исходника, упомянутых в доке (метрика полноты).
*   **[TRUTH]** — обратная проверка: символы, которые док утверждает в backticks, но которых НЕТ ни в исходнике, ни где-либо в проекте (ловит выдуманные методы/селекторы — напр. исторические `BaseBranch`, `eventUtils`, `withLoading`).

## Запуск
*   `python3 scripts/check_docs.py` — печатает отчёт; код возврата ≠0 при FAIL по LINKS/TREE/MIRROR.

## AI-контекст
*   Запускайте после ЛЮБЫХ правок документации/кода. `COMPLETE`/`TRUTH` — предупреждения для ручной проверки (возможны легитимные кросс-ссылки и доменные термины), `LINKS`/`TREE`/`MIRROR` — жёсткие инварианты сети. Дополняет генератор карты [generate_structure.py](generate_structure.md).
## Ключевые функции
*   `check_links()`, `check_tree()`, `check_mirror()` — жёсткие инварианты (LINKS/TREE/MIRROR).
*   `check_complete_and_truth()` — метрики COMPLETE/TRUTH; `source_symbols()`/`doc_claimed_tokens()` извлекают символы из исходника и заявленные доком; `all_docs()`/`all_sources()`/`all_linked_sources()`/`doc_source_map()` — индексация.
