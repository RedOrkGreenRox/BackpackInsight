# [Справочник данных (data.py)](../../../Backend/PlayerData/data.py)

## Назначение
Файл `data.py` является центральным хранилищем игровых констант, таблиц опыта и логики автоматического обнаружения версий данных.

---

## Ключевая логика

### 1. Авто-детект версии предметов (`get_latest_items_file`)
Скрипт сканирует директорию `Backend/DB/` на наличие файлов `items_*.json`. С помощью регулярных выражений извлекаются номера версий (Major, Minor, Patch), и выбирается файл с **наибольшим** номером. Это позволяет обновлять базу знаний простым добавлением нового JSON-файла.

### 2. Таблицы прогрессии
Файл содержит эталонные данные для расчетов:
*   **`PROFILE_EXP_NEED`**: Массив опыта для уровней игрока (100 уровней).
*   **`ITEM_LEVELING_CARDS`**: Требования по картам для улучшения предметов в зависимости от редкости (Common...Relic).
*   **`HERO_LEVELING_EXP`**: Требования к опыту для героев до 20 уровня.
*   **`PROFILE_AREAS`**: Пороги трофеев для открытия игровых зон.

### 3. Маппинг имен (`VALUES`)
Словарь для перевода технических ID в человекочитаемые имена (например, `Beekeeper` -> `Buzz`) и расшифровки технических ключей лога (`Mem` -> `Memory`).

---

## Связи (Dependencies)
*   Используется всеми моделями ([Hero](models/Hero.md), [Item](models/Item.md), [Profile](models/Profile.md)) и фабрикой [ProfileFactory](services/ProfileFactory.md) для проведения расчетов.

## AI-контекст
*   Использование `glob` и `max(key=extract_version)` делает систему управления данными гибкой: она не упадет, если в папке окажутся лишние JSON-файлы.
*   Функция `get_items` реализует паттерн Lazy Loading — данные загружаются в память только при первом обращении.
