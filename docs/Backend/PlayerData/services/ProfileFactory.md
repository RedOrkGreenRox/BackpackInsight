# [Фабрика профилей (ProfileFactory.py)](../../../../Backend/PlayerData/services/ProfileFactory.py)

## Назначение
`ProfileFactory` — это сложный сервис-оркестратор, отвечающий за преобразование "сырых" JSON-данных игрового лога в структурированные объекты базы данных с восстановлением всех связей.

---

## Ключевая логика

### 1. Система Кэширования Определений
Для ускорения парсинга фабрика использует агрессивное кэширование `ItemDefinition`:
*   **Двухуровневый индекс**: Поиск предмета идет сначала по `item_id`, а затем по отображаемому имени (`_definition_name_index`). Это решает проблему расхождения имен в логах и базе знаний.
*   **Thread-Safety**: Доступ к кэшу защищен реентерабельным локом (`_cache_lock`), что важно для многопоточной работы FastAPI.
*   **Preload**: При старте данные загружаются сначала из БД (актуальные), а затем дополняются из JSON-файлов (недостающие).

### 2. Парсинг Лога (`create_profile`)
Процесс разбит на этапы:
1.  **Валидация**: Проверка структуры JSON и наличия UID.
2.  **Технический блок**: Извлечение версии приложения и модели устройства.
3.  **Герои и Предметы**: Итерация по игровым словарям и создание связанных объектов.
4.  **Косметика**: Парсинг списка `UL` (Unlock List) с помощью регулярных выражений для извлечения скинов персонажей и баннеров.
5.  **Расчет статистики**: Вычисление суммарного опыта, уровня игрока и текущей игровой зоны на основе трофеев.

## Связи (Dependencies)
*   **Модели**: Порождает объекты [Profile](../models/Profile.md), [Hero](../models/Hero.md), [Item](../models/Item.md).
*   **Данные**: Использует константы уровней и опыта из [data.py](../data.md).

## AI-контекст
*   Метод `_calculate_level` содержит логику прогрессии игрока (таблица опыта), которая может меняться от версии к версии.
*   Фабрика спроектирована как **Stateful**, поэтому при использовании нескольких воркеров Uvicorn важно помнить, что у каждого будет свой экземпляр кэша.
