# [Модель Героя (Hero.py)](../../../../Backend/PlayerData/models/Hero.py)

## Назначение
Модель `Hero` описывает состояние конкретного героя в профиле игрока: его уровень, опыт, текущий рейтинг и лигу.

---

## Ключевая логика

### 1. Парсинг данных (`from_entry`)
Принимает список строк из игрового лога (например, `["Barbarian", "5", "150", "1200"]`) и преобразует его в объект:
*   **Имя**: Маппится через словарь `VALUES` (например, "Barbarian" -> "Harkon").
*   **Уровень**: В логах уровень 0-базовый, модель конвертирует его в 1-базовый.
*   **Престиж**: Если уровень > 20, вычисляется статус престижа.
*   **Лига**: Автоматически определяется на основе рейтинга (Bronze, Silver, Gold и т.д.).

### 2. Расчет опыта (`exp_req`)
Вычисляет количество опыта, необходимое для следующего уровня. До 20 уровня используются фиксированные значения из справочника, после 20 уровня применяется линейная прогрессия.

---

## Связи (Dependencies)
*   **БД**: Связан с [Profile](Profile.md) через Foreign Key `profile_id`.
*   **Данные**: Использует `HERO_LEVELING_EXP` и `HERO_LEAGUES` из [data.py](../data.md).

## AI-контекст
*   Поля `name` и `rating` помечены как индексируемые (`index=True`) для ускорения поиска и формирования будущих лидербордов.
*   Метод `__str__` содержит логику отрисовки ASCII-прогрессбара опыта, что полезно для отладки в консоли.
