# [Модель Профиля (Profile.py)](../../../../Backend/PlayerData/models/Profile.py)

## Назначение
Модель `Profile` — это центральный узел данных игрока. Она объединяет аналитические поля (для SQL-запросов), техническую информацию об устройстве и ссылки на героев и предметы.

---

## Структура данных

### 1. Аналитические поля (SQL)
Поля типа `nickname`, `trophies`, `level`, `coins`, `user_id` вынесены в отдельные колонки БД с индексами. Это позволяет строить глобальную статистику и быстро находить профили.

### 2. Техническая и Игровая информация (JSON)
Сырые данные из лога сохраняются в виде JSON-блоб (`technical_info_data`, `game_info_data`). 
*   **`TechnicalInfo`**: Схема для данных об ОС, версии приложения, модели устройства и памяти.
*   **`GameInfo`**: Схема для валют, истории покупок, разблокированных баннеров и скинов.

### 3. Таймстампы
Поля `created_at` и `updated_at` используются для:
*   Семантики **Upsert** (обновление существующего профиля).
*   Автоматической очистки старых данных (TTL) через планировщик.

---

## Ключевая логика
*   **`from_json`**: Статический метод-фабрика, делегирующий создание объекта классу [ProfileFactory](../services/ProfileFactory.md).
*   **`to_frontend_view`**: Метод сериализации, который собирает данные из всех связанных таблиц (Hero, Item) в один плоский JSON-ответ для веб-интерфейса.

## Связи (Dependencies)
*   **Отношения**: "Один-ко-многим" с [Hero](Hero.md) и [Item](Item.md).

## AI-контекст
*   Проперти `technical_information` и `game_information` создают Pydantic-объекты "на лету" из JSON-данных. Это обеспечивает типизированный доступ к полям без конфликтов с состоянием SQLAlchemy.
