# [API Бэкенда (api.py)](../../../Backend/PlayerData/api.py)

## Назначение
Файл `api.py` является точкой входа для FastAPI-приложения. Он объединяет логику маршрутизации, защиты API, управления сессиями БД и автоматической синхронизации данных.

---

## Ключевая функциональность

### 1. Безопасность и Защита
*   **Proxy Secret**: Проверка заголовка `X-Internal-Secret` через `hmac.compare_digest` для предотвращения прямого доступа к API в обход прокси.
*   **Rate Limiting**: Используется `slowapi` для ограничения частоты запросов к эндпоинту `/api/profile` (по умолчанию 20 зап/мин).
*   **CORS**: Настройка разрешенных источников через переменную `CORS_ORIGIN`.

### 2. Жизненный цикл и Синхронизация (`lifespan`)
При старте приложения выполняется `_sync_static_data`:
*   Вычисляет SHA-256 хеш актуальных JSON-файлов предметов.
*   Сравнивает его с хешем в БД.
*   Если данные обновились, происходит **полная пересинхронизация**: очистка старых определений и загрузка новых. Это гарантирует, что база знаний API всегда соответствует последней версии игры.

### 3. Обработка профилей (`/api/profile`)
*   Принимает JSON игрового лога.
*   **Upsert Logic**: Если у игрока уже есть запись в БД (по `user_id`), старый снимок удаляется (включая связанных героев/предметы) и создается новый. Это предотвращает дублирование данных одного игрока.
*   Возвращает структурированный ответ, готовый для отображения на фронтенде.

## AI-контекст
*   Лимиты запросов (`limiter`) активируются только при наличии `API_SECRET` в окружении, что удобно для локальной разработки.
*   Для PostgreSQL при старте создаются GIN-индексы для нечеткого поиска, если они не были созданы миграциями.
