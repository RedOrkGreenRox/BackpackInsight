# [Атрибуты Git (.gitattributes)](../.gitattributes)

## Назначение
Файл `.gitattributes` используется для настройки того, как Git должен обрабатывать определенные файлы в репозитории (переносы строк, бинарные данные, подсчет статистики GitHub).

---

## Ключевые настройки

### 1. Нормализация окончаний строк (`eol=lf`)
*   **Проблема**: Windows использует CRLF (`\r\n`), а Linux/macOS — LF (`\n`). Это часто ломает скрипты и конфигурационные файлы при деплое.
*   **Решение**: Для файлов `.env` и всех текстовых файлов принудительно установлен формат **LF**. Это критично для корректного чтения секретов бэкендом внутри Docker-контейнера.

### 2. Обработка бинарных файлов
*   Изображения (`.png`, `.jpg`) помечены как `binary`, что предотвращает попытки Git «нормализовать» их содержимое, что привело бы к порче файлов.

### 3. Статистика Linguist (GitHub)
*   **`linguist-generated=true`**: Все `.json` файлы помечены как сгенерированные. Это скрывает огромные файлы данных предметов из общей статистики языков проекта, чтобы JSON-данные не перекрывали реальный код на TypeScript/Python.
*   **`linguist-vendored`**: Сторонние библиотеки и системные файлы (Dockerfile, .bat) также исключены из статистики кода.

---

## AI-контекст
*   Если GitHub показывает, что ваш проект на 90% состоит из JSON — проверьте настройки `linguist-generated` в этом файле.
*   При возникновении ошибок "Invalid character in password" на сервере, убедитесь, что `.env` файл был сохранен с окончаниями строк LF (за что и отвечает этот конфиг).
