# Синтаксис поиска и строгих тегов

## Обычный текст

Обычный ввод в строку поиска работает как мягкий fuzzy/semantic search.

Примеры:

```text
gambler
ядовитый
кинжал яд
```

Для обычного текста используется Fuse + терминологическое расширение. То есть `ядовитый` может найти предметы с `Poison`.

## Строгие теги

Теги пишутся в квадратных скобках, как в описаниях предметов:

```text
[Armor]
[Poison]
[CritChance]
[MeleeWeapon]
```

Строгие теги НЕ используют семантику. Это фильтр по конкретному игровому тегу/иконке/термину.

## OR внутри одной группы

Несколько тегов внутри одних `[]` работают как ИЛИ:

```text
[MeleeWeapon RangedWeapon]
```

Значит:

```text
MeleeWeapon ИЛИ RangedWeapon
```

Можно разделять пробелами, запятыми, `|` и другими небуквенными символами:

```text
[MeleeWeapon|RangedWeapon]
[Poison, Burn, Bleed]
```

## AND между группами

Несколько отдельных групп работают как И:

```text
[Poison] [MeleeWeapon RangedWeapon]
```

Значит:

```text
Poison И (MeleeWeapon ИЛИ RangedWeapon)
```

## Смешивание текста и тегов

Обычный текст и строгие теги можно комбинировать:

```text
dagger [Poison]
```

Значит:

```text
мягкий поиск по `dagger`
И строгий фильтр `Poison`
```

Еще пример:

```text
ядовитый [MeleeWeapon RangedWeapon]
```

Значит:

```text
мягкий semantic search по `ядовитый`
И строгий фильтр по типу: MeleeWeapon ИЛИ RangedWeapon
```
