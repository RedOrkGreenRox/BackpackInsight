# Поиск на странице «Предметы»

Поиск устроен в два слоя: обычный визуальный режим и продвинутый ручной синтаксис.

## Обычный режим

Строка поиска сверху — это только текстовый поиск по уже отфильтрованному списку.

```text
нож
череп
яд
sword
```

Конкретные условия выбираются кнопками в панели фильтров. Все категории по умолчанию раскрыты.

### Списки «Искать» и «Исключить»

Списки находятся вверху панели фильтров:

- **Искать** — предмет должен иметь выбранные теги.
- **Исключить** — предмет не должен иметь выбранные теги.

Кнопка-тег работает циклом:

1. первый клик — зелёный тег, добавить в **Искать**;
2. второй клик — красный тег, добавить в **Исключить**;
3. третий клик — вернуть тег в нейтральное состояние.

Теги в этих списках всегда точные: под капотом они работают как `<Тег>`.

Пример:

```text
Искать: Тип Knife, Тип Dagger
Исключить: Purchasable
```

логически равно:

```text
([<Knife>] | [<Dagger>]) & ![<Purchasable>]
```

Внутри одной категории несколько тегов работают как **ИЛИ**, между категориями — **И**.

## Сортировка в обычном режиме

Сортировки задаются в категории «Сортировка» тремя приоритетами:

- Релевантность;
- Редкость;
- Алфавит.

Клик по сортировке меняет направление:

```text
не выбрана -> по убыванию -> по возрастанию -> не выбрана
```

Маленькие стрелки на кнопке сортировки двигают её выше/ниже в списке приоритетов.

## Справка `?`

Кнопка `?` находится рядом со списками «Искать/Исключить» и доступна уже в обычном режиме. В ней есть примеры с кнопками копирования и полный список тегов из текущей базы.

## Продвинутые настройки

Переключатель **«Продвинутые настройки»** находится внизу панели фильтров. После включения появляются:

- логический раздел `[ ]`, `И`, `ИЛИ`, `НЕ` — первым после списков;
- формула, собранная из списков «Искать/Исключить»;
- ручной язык запросов в строке поиска.

Ручной ввод удобен, если разобраться: сложный запрос можно написать быстрее, чем собирать кнопками.

## Продвинутый синтаксис

### Обычный текст

```text
sword
кинжал яд
череп
```

Это fuzzy-поиск с алиасами.

### Умный НЕ точный тег

```text
[Poison]
[Armor]
[MeleeWeapon]
```

Такой тег ищет шире: по описаниям, игровым иконкам, терминам и алиасам.

### Конкретный точный тег

```text
[<Knife>]
[<Dagger>]
[<Legendary>]
[<Chana>]
[<Purchasable>]
```

`<...>` означает точный тег. Для типов вроде Knife/Dagger лучше использовать именно его.

### И, ИЛИ, НЕ

```text
&    И    AND
|    ИЛИ  OR
!    НЕ   NOT
```

Примеры:

```text
[Poison] & [<Dagger>]
[<Knife>] | [<Dagger>]
![<Purchasable>]
([Poison] | [Burn]) & ![<Boon>]
```

### Сравнения

Поддерживаются одиночные и двойные сравнения:

```text
damageMax > 10
cooldown < 2
1 < cooldown < 5
accuracy >= 80
cost <= 5
```

Понимаемые поля/алиасы:

```text
damageMin, minDamage, damage_min
damageMax, maxDamage, damage_max
accuracy, acc
criticalChance, critChance, critical_chance
criticalDamage, critDamage, critical_damage
staminaCost, stamina
cooldown, cd
coinValue, cost, gold, price, value
level, lvl
```

### Текстовая сортировка

```text
{relevance}
{rarity down}
{rarity up}
{alphabet up}
{alphabet down}
```

## Алиасы

Русские слова могут расширяться в игровые термины:

```text
нож -> knife / dagger
кинжал -> dagger
меч -> sword
череп -> skull
яд -> poison
броня -> armor
крит -> critical
урон -> damage
```

Для точного результата лучше выбирать тег кнопкой или писать точный тег:

```text
[<Skull>]
[<Knife>]
[<Dagger>]
```
