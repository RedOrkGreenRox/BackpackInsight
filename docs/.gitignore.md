# [Исключения Git (.gitignore)](../.gitignore)

## Назначение
Файл `.gitignore` предотвращает попадание временных, технических и конфиденциальных файлов в систему контроля версий Git. Это поддерживает чистоту репозитория и предотвращает утечку секретов.

---

## Основные правила

### 1. Безопасность (Secrets)
*   **`.env`**, `.env.local` и др. — файлы с паролями, API-ключами и настройками БД. Они **никогда** не должны быть закоммичены.
*   Конфиги ботов и списки вайтлистов (`whitelist.json`).

### 2. Среда выполнения (Environment)
*   `__pycache__/`, `*.pyc` — скомпилированный байт-код Python.
*   `node_modules/` — зависимости Node.js (занимают много места, устанавливаются через `npm install`).
*   `venv/`, `.venv/` — виртуальные окружения Python.

### 3. Результаты сборки (Build Artifacts)
*   `dist/` — собранные файлы фронтенда.
*   `*.db`, `*.sqlite3` — локальные файлы баз данных. В репозитории должна храниться структура (SQL/Миграции), но не сами данные.

### 4. Мусор ОС и IDE
*   `.vscode/`, `.idea/` — настройки редакторов кода.
*   `.DS_Store` (macOS), `Thumbs.db` (Windows).

---

## AI-контекст
*   Если вы создали новый тип конфига или временную папку для логов, убедитесь, что она описана в этом файле.
*   Игнорирование `.db` файлов в папках `Backend/DB` и `Frontend/Web` критически важно для предотвращения конфликтов при слиянии веток.
