# [Модель героя (Hero.py)](../../../../Backend/PlayerData/models/Hero.py)

## Назначение
Этот файл описывает структуру данных героя в базе данных и инкапсулирует логику расчета прогресса: уровней, престижа, требуемого опыта и лиг.

## Связи (Dependencies)
*   **Типы данных**: Основан на `SQLModel`.
*   [Данные прогрессии](../data.md): Использует справочники `HERO_LEVELING_EXP` и `HERO_LEAGUES`.
*   [Модель профиля](./Profile.md): Связан отношением «Многие к одному» (много героев в одном профиле).
*   **Константы**: Использует словарь `VALUES` из `data.py` для перевода технических имен героев в игровые.

---

## Подробное описание методов и свойств

### `classmethod from_entry(entry: List[str])` — **ГЛУБОКАЯ ЛОГИКА ПАРСИНГА**
*   **Задача**: Создать объект героя из строки игрового лога.
*   **Формат лога**: `["Barbarian", "5", "150", "1200"]`.
*   **Логика уровня**: В JSON уровень хранится от 0. Метод прибавляет 1 для человекочитаемости.
*   **Логика престижа**: В **Backpack Brawl** престиж начинается после 20 уровня. Если уровень > 20, метод вычитает 20 и устанавливает флаг `prestige = True`.
*   **Логика лиги**: Вычисляет лигу по формуле `рейтинг // 500`.

### `property exp_req`
*   **Задача**: Вычислить опыт до следующего уровня.
*   **Логика**: До 20 уровня берет данные из таблицы. После 20-го — переключается на линейную формулу прогрессии: `1000 + 100 * (уровень - 21)`, но не более 3000 XP.

### `to_frontend_view()`
*   **Задача**: Подготовить объект для отправки на фронтенд.
*   **Связь**: Этот словарь должен соответствовать [Интерфейсу героя](../../../frontend/ground/branches/profile/_profile/utils/profile-types.md).

## AI-контекст
*   **Имена**: Обратите внимание, что в логе герои часто называются техническими именами (например, `Barbarian`), которые метод `from_entry` автоматически конвертирует в игровые (`Harkon`) через словарь `VALUES`.
*   **База данных**: Поля `name` и `rating` имеют индексы для ускорения работы поиска и глобальных таблиц лидеров.
