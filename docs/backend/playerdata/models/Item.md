# [Модели предметов (Item.py)](../../../../Backend/PlayerData/models/Item.py)

## Назначение
Этот файл описывает "скелет" экономики **Backpack Brawl**. Он разделяет статическую информацию о предметах (база знаний) и динамическую информацию об инвентаре игрока.

## Архитектура: Разделение моделей
В проекте используется двухуровневая структура предметов:

1.  **[Определение предмета (ItemDefinition)](#itemdefinition)**: Глобальная статика. Описывает, что такое «Меч» вообще (урон, цена, рецепты).
2.  **[Экземпляр предмета (Item)](#item)**: Персональные данные игрока. Описывает «Меч» конкретного пользователя (его уровень, количество собранных карточек).

---

## Подробное описание классов

### `ItemDefinition`
*   **Связь**: Напрямую соответствует записи в `items.json`.
*   **item_id**: Строковый идентификатор, являющийся первичным ключом.
*   **Геометрия**: Хранит поля `item_shape` и `item_stars` в формате JSON для отрисовки сетки рюкзака.
*   **Рецепты**: Список ингредиентов для крафта.

### `Item`
*   **Связь**: Привязан к [Модели профиля](./Profile.md) через `profile_id`.
*   **Прокси-свойства**: Использует декоратор `@definition_proxy_property` из [Утилит](../../../../Backend/PlayerData/utils.py). Это позволяет обращаться к `item.name` или `item.rarity` напрямую, хотя эти данные физически лежат в связанной таблице `ItemDefinition`.
*   **`cards_need` (property)**: Вычисляет порог для следующего уровня, используя справочник `ITEM_LEVELING_CARDS`. Возвращает `-1`, если предмет достиг максимального уровня.
*   **`total_xp` (property)**: Рассчитывает суммарный опыт, который этот предмет принес в общую копилку игрока.

## Ключевая логика: Оптимизация и Fallback
Метод `_get_definition()` реализует стратегию отказоустойчивости:
1.  Сначала пытается взять данные через ORM-связь `SQLModel`.
2.  Если связь отсутствует (например, объект отвязан от сессии), он обращается к внутреннему кешу [Фабрики профилей](../services/ProfileFactory.md). Это критично для высокой производительности при массовом парсинге сотен предметов.

## AI-контекст
*   **Типы**: Обратите внимание на `sa_column=Column(JSON)`. Это позволяет хранить сложные вложенные структуры (координаты звезд, статы) прямо в PostgreSQL без создания десятков лишних таблиц.
*   **Интеграция**: Метод `to_frontend_view()` должен синхронно обновляться с [Интерфейсом предмета](../../../frontend/ground/branches/profile/_profile/utils/profile-types.md) на фронтенде.
