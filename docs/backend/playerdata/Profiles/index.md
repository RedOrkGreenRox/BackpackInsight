# [Эталонные профили (Profiles/*.json)](../../../../Backend/PlayerData/Profiles/)

## Назначение
Папка `Profiles/` содержит коллекцию реальных JSON-логов игроков игры Backpack Brawl. Эти файлы используются как эталонные данные (gold standards) для тестирования и отладки парсеров бэкенда.

---

## Список профилей и их роль

В папке находятся 13 файлов, представляющих различные игровые сценарии:
*   **DI4.json / DI4-new.json**: Профили для проверки миграции данных и обновления существующих записей.
*   **German.json / Sky.json / Marat.json**: Профили опытных игроков с большим количеством предметов и прокачанными героями (используются для нагрузочного тестирования парсинга).
*   **Sky_of_BFG.json / Teger.json / Molodoy_Zhuk.json**: Профили с редкими косметическими предметами (скины, баннеры) для проверки логики `_parse_unlockables`.
*   **hush.json / merul.json / rimaster.json / xr1stos422.json**: Дополнительные наборы данных для верификации консистентности опыта (XP Consistency).

---

## Применение в системе
1.  **Интеграционное тестирование**: Скрипт [test_profiles_integration.py](../../../../tests/test_profiles_integration.py) автоматически подхватывает все файлы из этой папки и прогоняет через них парсер `ProfileFactory`.
2.  **Регрессия**: При добавлении новых функций в бэкенд (например, расчет рейтинга лиг), эти файлы позволяют убедиться, что старая логика не сломалась.
3.  **Демонстрация**: Файлы могут использоваться для наполнения локальной базы данных демонстрационными данными.

## AI-контекст
*   Файлы содержат BOM (Byte Order Mark) в кодировке UTF-8, что было учтено при исправлении тестов ([подробнее здесь](../../../../tests/test_profiles_integration.py)).
*   Эти данные являются анонимизированными снимками игрового состояния и не содержат чувствительной личной информации, кроме игровых никнеймов и UID Unity.
