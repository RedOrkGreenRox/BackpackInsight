# [Фабрика профилей (ProfileFactory.py)](../../../Backend/PlayerData/services/ProfileFactory.py)

## Назначение
Центральный сервис бизнес-логики. Он инкапсулирует весь процесс трансформации "сырого" JSON лога игры **Backpack Brawl** в полноценную объектную модель базы данных. Это самый сложный узел бэкенда, отвечающий за расчеты статов, парсинг скинов и управление кешем.

## Связи (Dependencies)
*   **Модели**: Импортирует и создает объекты [Profile](../models/Profile.md), [Hero](../models/Hero.md) и [Item](../models/Item.md).
*   [Константы](../constants.md): Использует словари ключей для навигации по JSON.
*   [Данные](../data.md): Использует справочники опыта для расчетов.

## Подробное описание методов

### Управление кешем
Для обеспечения экстремальной производительности парсинга, Фабрика использует внутренний кеш определений предметов `_definition_cache`.
*   `preload_definitions(session)`: Загружает все предметы из БД в память при старте. Если БД пуста — дополняет из JSON.
*   `refresh_from_database()`: Обновляет кеш без перезагрузки сервера, используя метод `expunge` для отвязки объектов от сессии.

### `create_profile(json_data)` — **ГЛАВНЫЙ ПРОЦЕСС**
1.  **Валидация**: Проверяет структуру JSON через `_validate_json`.
2.  **Тех. инфо**: Извлекает версию приложения, ID устройства и ОС.
3.  **Герои**: Проходит по секции `Hero`, создавая объекты через `Hero.from_entry`.
4.  **Предметы**: Проходит по секции `Item`, связывая их со статической базой знаний.
5.  **Косметика**: Парсит поле `UL` (Unlock List) для поиска разблокированных скинов и баннеров.
6.  **Расчеты**: Вызывает `_calculate_stats` для вычисления итогового уровня и зоны арены.

### Глубинные расчеты
*   `_calculate_level(total_xp)`: Алгоритм обхода массива `PROFILE_EXP_NEED`. Если опыт превышает массив, переключается на формулу бесконечной прогрессии.
*   `_calculate_area(trophy)`: Маппинг рейтинга игрока на ID зоны арен (01-20).
*   `_calculate_item_stats()`: Группирует коллекцию игрока по редкостям для отображения в шапке профиля.

### Парсинг косметики
*   `_parse_skins()`: Использует регулярное выражение `^([A-Za-z]+)Skin([A-Za-z]+)$` для извлечения пары Герой-Скин из общего списка разблокировок. Это позволяет фронтенду знать, какие кнопки переключения скинов показывать.

## AI-контекст
*   **Циклические зависимости**: Обратите внимание, что многие импорты моделей происходят внутри методов. Это сделано специально, чтобы избежать конфликтов в Python.
*   **Производительность**: Кеш и статичные методы позволяют Фабрике обрабатывать профиль с 500+ предметами менее чем за 50мс.
*   **Расширяемость**: Если в игре появятся новые типы разблокировок (например, "Питомцы"), логику их парсинга нужно добавлять в метод `_parse_unlockables`.
