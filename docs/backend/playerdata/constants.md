# [Константы парсинга (constants.py)](../../../Backend/PlayerData/constants.py)

## Назначение
Этот файл содержит словари-маппинги для всех ключей JSON, используемых в файлах профиля **Backpack Brawl**. Он избавляет код от «магических строк», обеспечивая единую точку правды для именования полей.

## Связи (Dependencies)
*   [Сервис фабрики профилей](services/ProfileFactory.md): Главный потребитель. Использует эти константы для извлечения данных из сырого JSON.
*   [Модели данных](./models/Profile.md): Использует дефолтные значения при создании объектов.

---

## Описание словарей

### `TECHNICAL_KEYS`
Содержит сокращенные ключи технической информации об устройстве и версии игры (например, `AV` -> "Actual Version", `UID` -> "User ID"). 

### `GAME_KEYS`
Ключи игровых данных:
*   `PH`: История покупок.
*   `UL`: Список разблокированного контента (скины, предметы).
*   `TROPHY`: Текущий рейтинг.
*   `HERO`: Секция данных о героях.
*   `ITEM`: Секция данных о предметах.

### `DEFAULT_VALUES`
Набор базовых значений для новых или неполных записей (Rarity: Common, Level: 1 и т.д.).

## AI-контекст
*   **Гигиена кода**: При добавлении парсинга новых полей из игрового лога, сначала добавьте ключ сюда, а затем используйте его в коде.
*   **Синхронизация**: Эти константы должны соответствовать тому, что игра записывает в файлы сохранений. Если разработчики игры изменят `PH` на `PurchaseHistory`, изменения нужно внести только в этот файл.
