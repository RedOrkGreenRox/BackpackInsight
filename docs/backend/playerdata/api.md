# [API Бэкенда (api.py)](../../../Backend/PlayerData/api.py)

## Назначение
Главный файл веб-сервера. Здесь сосредоточены настройки безопасности, CORS, маршруты (endpoints) и логика синхронизации статических данных при запуске.

## Связи (Dependencies)
Оркестрирует работу всех систем бэкенда:
*   [Модели](./models/Profile.md): Для сохранения и извлечения профилей.
*   [БД](../../backend/db/database.md): Использует `engine` и `get_session`.
*   [Фабрика профилей](./services/ProfileFactory.md): Для глубокого парсинга входящих JSON.
*   [Загрузчик данных](./data.md): Для первичного импорта справочников.

## Подробное описание функций

### Безопасность (`verify_proxy_secret`)
*   **Задача**: Защита прямого доступа к API.
*   **Логика**: Проверяет заголовок `X-Internal-Secret`. Использует `hmac.compare_digest` для защиты от timing-атак. Если секрет не совпадает, сервер возвращает 403 Forbidden.

### Жизненный цикл (`lifespan`)
*   **Синхронизация**: При старте сервера вызывает `_sync_static_data`. Это гарантирует, что база данных всегда соответствует JSON-файлам из папки `DB/`.

### Синхронизация данных (`_sync_static_data`) — **СЛОЖНАЯ ЛОГИКА**
*   **Алгоритм хеширования**:
    1.  Генерирует SHA-256 хеш текущего содержимого JSON-файла предметов.
    2.  Сравнивает его с хешем предметов в БД.
    3.  **Перезагрузка**: Если хеши не совпадают (данные обновились), сервер полностью очищает таблицы `ItemDefinition` и `Item` и загружает новые данные. Это предотвращает конфликты версий.
*   **Отказоустойчивость**: Очистка старых данных — критичная операция. При ошибке БД (`SQLAlchemyError`) ресинк прерывается с откатом транзакции, чтобы избежать дубликатов PK и нарушения ссылочной целостности (FK).

### Маршруты (Endpoints)
*   `GET /api/items`: Возвращает справочник предметов. Поддерживает `limit` и `offset` для пагинации на фронтенде.
*   `POST /api/profile`: 
    *   Принимает JSON лога игрока.
    *   Проверяет размер запроса (ограничение 1MB для защиты от DoS).
    *   **Rate-limiting**: Эндпоинт защищён лимитом (`RATE_LIMIT_PROFILES`, по умолчанию `20/minute` через `slowapi`), включается только при заданном `API_SECRET`.
    *   **Upsert по `user_id`**: Если профиль с таким UID уже существует — он заменяется (старые герои/предметы удаляются, вставляется свежий снимок). Один игрок — одна строка.
    *   Вызывает [Фабрику профилей](./services/ProfileFactory.md) для анализа и возвращает результат, готовый для отрисовки.

### Политика хранения (`cleanup_old_profiles`)
*   Удаляет профили (и связанные героев/предметы) старше `PROFILE_RETENTION_DAYS` дней (`0` = отключено). Должна запускаться внешним планировщиком (cron). Решает проблему бесконечного накопления PII.

## AI-контекст
*   **CORS**: Разрешенные домены берутся из переменной `CORS_ORIGIN`. По умолчанию настроен на продакшн Cloudflare.
*   **Индексы PostgreSQL**: GIN-индексы для триграммного поиска (`pg_trgm`) создаются **миграцией Alembic `0002`** сразу после `alembic upgrade head`. Метод `create_indexes` выполняется как no-op на SQLite и безопасно дублирует создание индексов на PostgreSQL при локальном запуске без миграций.
