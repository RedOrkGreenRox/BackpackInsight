# [Утилиты (utils.py)](../../../Backend/PlayerData/utils.py)

## Назначение
Файл `utils.py` содержит вспомогательные функции и декораторы, предназначенные для уменьшения дублирования кода и обеспечения безопасного доступа к вложенным данным.

---

## Ключевые функции

### 1. Декоратор `definition_proxy_property`
Обеспечивает связь между динамическим объектом `Item` и его статическим описанием `ItemDefinition`. 
*   **Логика**: Позволяет обращаться к полям описания (name, rarity, combat_stats) напрямую через объект предмета игрока. Если описание не найдено, возвращает заданное значение по умолчанию.

### 2. `safe_get_nested`
Утилита для безопасного извлечения данных из глубоко вложенных словарей. Предотвращает ошибки `KeyError`, возвращая `default`, если какой-либо ключ в цепочке отсутствует.

### 3. `create_item_definition_safe`
Фабричная функция для создания объектов `ItemDefinition` с базовой валидацией (проверка на пустые строки в ID и именах).

---

## Связи (Dependencies)
*   **Используется в**: Модели [Item](models/Item.md) для реализации прокси-свойств.

## AI-контекст
*   Использование декораторов для прокси-свойств значительно сокращает объем модели `Item.py`, делая её более читаемой.
*   При расширении модели профиля (например, для глубокого вложенного конфига), рекомендуется использовать `safe_get_nested`.
