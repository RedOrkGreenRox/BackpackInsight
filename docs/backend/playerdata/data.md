# [Сервис справочника предметов (data.py)](../../Backend/PlayerData/data.py)

## Назначение
Этот файл отвечает за загрузку статических данных игры из JSON-файлов. Он является мостом между файловой системой сервера и базой данных PostgreSQL, обеспечивая актуальность характеристик предметов **Backpack Brawl**.

## Связи (Dependencies)
*   **Файлы данных**: Работает с `items_*.json` в директории `Backend/DB/`.
*   [Модель предмета](./models/Item.md): Использует константы опыта и карточек для расчетов.
*   [API бэкенда](./api.md): Вызывает синхронизацию при старте сервера.

---

## Подробное описание функций

### `get_latest_items_file()` — **АВТОМАТИЗАЦИЯ**
*   **Задача**: Найти файл с самой свежей версией данных.
*   **Логика**: Использует `glob` для поиска всех файлов `items_*.json` и регулярное выражение для извлечения версии (Major, Minor, Patch).
*   **Результат**: Возвращает путь к файлу с наибольшим номером версии. Это позволяет обновлять базу предметов просто закидывая новый файл в папку `DB/`.

### `load_items()`
*   **Логика**: Выполняет чтение и парсинг найденного файла.
*   **Отказоустойчивость**: Содержит цепочку `try/except` для обработки битых JSON-файлов. Если файл не найден, возвращает пустой список, предотвращая падение всего сервера.

### `get_items()`
*   **Оптимизация**: Реализует паттерн **Lazy Loading**. Список предметов загружается в память и индексируется по ID только при первом обращении.

## Справочники (Массивы данных)

### `PROFILE_EXP_NEED`
Массив, содержащий требования опыта для каждого уровня аккаунта игрока. Именно на этот массив ссылается [Фабрика профилей](./services/ProfileFactory.md) при расчете прогресса.

### `ITEM_LEVELING_CARDS` / `ITEM_LEVELING_EXP`
Словари, описывающие прогрессию предметов в зависимости от их редкости. 
*   **Важно**: Разные редкости (Epic, Mythic) имеют разную кривую прогрессии и разные "потолки" уровней.

## AI-контекст
*   **Обновления**: Это "сердце" версионности проекта. Если после обновления игры Backpack Brawl изменились характеристики предметов, проверьте наличие нового JSON-файла и работу метода `extract_version`.
*   **Производительность**: Кеширование в `_items_list` экономит секунды при парсинге профилей с большим количеством предметов.
