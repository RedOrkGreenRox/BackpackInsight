# [Статические данные предметов (items_*.json)](../../../Backend/DB/)

## Назначение
Серия файлов `items_X_X_X.json` представляет собой снимки (snapshots) базы знаний игры **Backpack Brawl** для конкретных версий игрового клиента. 

Доступные версии:
*   `items_3_1_0.json`
*   `items_4_0_0.json`
*   `items_5_0_0.json`

## Структура данных
Каждый файл содержит массив объектов предметов со следующими ключевыми полями:
*   `id`: Уникальный идентификатор (например, `Wooden Sword`).
*   `name`: Локализованное имя.
*   `rarity`: Редкость (Common, Rare, Epic, Legendary, Godly).
*   `itemTypes`: Категории (Weapon, Melee, Food и т.д.).
*   `recipes`: Список комбинаций для создания этого предмета.
*   `combatStats`: Числовые параметры урона, скорости атаки и выносливости.
*   `itemShape`: Координаты сетки инвентаря, которые занимает предмет.

---

## Использование в проекте
1.  **Синхронизация**: При запуске бэкенда логика в [api.py](../playerdata/api.md) сравнивает хеш этих файлов с данными в таблице `ItemDefinition`.
2.  **Обновление**: При выходе новой версии игры достаточно положить новый JSON в эту папку. Система автоматически выберет файл с самой высокой версией (логика в [data.py](../playerdata/data.md)).

## AI-контекст
*   Файлы имеют большой объем (до 3Мб), поэтому они помечены в `.gitattributes` как `linguist-generated`, чтобы не искажать статистику языков в репозитории.
*   При ручном редактировании JSON важно сохранять структуру полей, иначе `ProfileFactory` не сможет корректно привязывать предметы к профилям игроков.
