# [Конфигурация базы данных (database.py)](../../../Backend/DB/database.py)

## Назначение
Этот файл является ядром слоя доступа к данным (DAL). Он отвечает за обнаружение настроек окружения, инициализацию движка базы данных (PostgreSQL или SQLite) и управление сессиями через [SQLModel](https://sqlmodel.tiangolo.com/).

## Связи (Dependencies)
*   **Окружение**: Использует `.env` файл для получения секретов (пароли, хосты).
*   **Использование**: 
    *   [API эндпоинты](../playerdata/api.md): Импортируют `get_session` для выполнения запросов.
    *   [Модели данных](../playerdata/models/Profile.md): Используют этот движок для миграций и операций сохранения.

---

## Подробное описание функций и логики

### Поиск конфигурации (`find_env_file`)
*   **Сложность**: Реализует рекурсивный поиск файла `.env`.
*   **Логика**: Проверяет текущую папку, родительскую и корень проекта. Это гарантирует работу бэкенда как внутри Docker-контейнера, так и при локальном запуске скриптов из разных директорий.

### Инициализация движка (`engine`)
*   **Гибридный режим**: 
    *   По умолчанию пытается подключиться к **PostgreSQL**.
    *   Если установлена переменная `USE_SQLITE=True` или подключение к Postgres не удалось, автоматически переключается на **SQLite**.
*   **Безопасность**: Использует `os.getenv` с безопасными дефолтами для разработки, но позволяет жестко задать параметры для продакшена.

### Управление сессиями (`get_session`)
*   **Логика**: Генератор (`yield`), который открывает сессию в контекстном менеджере.
*   **Зачем**: Гарантирует автоматическое закрытие соединения с БД после завершения обработки API-запроса, предотвращая утечки ресурсов.
