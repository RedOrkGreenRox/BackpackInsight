# [Конфигурация базы данных (database.py)](../../../Backend/DB/database.py)

## Назначение
Файл `database.py` является центральным узлом слоя доступа к данным (DAL). Он управляет жизненным циклом подключения к базе данных, переключением между СУБД (PostgreSQL/SQLite) и поиском конфигурационных файлов.

---

## Ключевая логика

### 1. Универсальный поиск `.env` (`find_env_file`)
Реализует алгоритм рекурсивного поиска файла настроек окружения. Проверяет три уровня:
*   Текущая директория (`Backend/DB/`)
*   Родительская директория (`Backend/`)
*   Корень проекта

Это критично для корректной работы бэкенда при разных способах запуска (Docker, локальный запуск через `uvicorn` или запуск тестов).

### 2. Динамический выбор движка (Engine)
Проект поддерживает гибридный режим работы:
*   **PostgreSQL**: Основная БД для продакшена и Docker-среды.
*   **SQLite**: Автоматический фолбэк для локальной разработки или если переменная `USE_SQLITE` установлена в `True`. 
    *   *Путь к SQLite*: `/Frontend/Web/database.db` (сделано для удобства отладки фронтенда).

### 3. Инъекция сессий (`get_session`)
Генератор сессий для FastAPI. Использует контекстный менеджер `with Session(engine)`, что гарантирует:
*   Автоматическое закрытие соединения.
*   Предотвращение утечек пула подключений.
*   Возврат (`yield`) сессии в зависимости эндпоинта.

## Связи (Dependencies)
*   **Использует**: `sqlmodel` для создания `engine`, `python-dotenv` для загрузки настроек.
*   **Используется в**:
    *   [api.py](../playerdata/api.md) — для получения сессий в эндпоинтах.
    *   [bootstrap.py](bootstrap.md) — для проверки состояния таблиц.
    *   [migrations/env.py](migrations/env.md) — Alembic берет `engine` именно отсюда.

## AI-контекст
*   При возникновении ошибок "Database is locked" (в режиме SQLite), убедитесь, что `connect_args={"check_same_thread": False}` не был удален.
*   Переменная `DATABASE_URL` строится динамически из компонентов (User, Password, Server), что упрощает настройку через Docker Compose.
