# [Начальная схема БД (0001_initial_schema.py)](../../../../../Backend/DB/migrations/versions/0001_initial_schema.py)

## Назначение
Это первая (базовая) ревизия базы данных BackpackInsight. Она создает фундамент для хранения всех игровых данных.

---

## Создаваемые таблицы

### 1. `itemdefinition`
Справочник всех предметов в игре. Хранит статические данные: рецепты, статы, редкости.
*   **PK**: `item_id` (строка).
*   **Индексы**: по редкости и привязке к герою.

### 2. `profile`
Снимки профилей игроков.
*   **Поля**: никнейм, трофеи, уровень, монеты, гемы, данные об устройстве.
*   **Индексы**: по уровню, никнейму и `user_id`.

### 3. `hero`
Герои, принадлежащие конкретному профилю.
*   **Связь**: Foreign Key на `profile.pk`.

### 4. `item`
Инстансы предметов в инвентаре игрока.
*   **Связи**: FK на `profile.pk` и FK на `itemdefinition.item_id`.
*   Хранит текущий уровень предмета и количество карточек.

---

## Роль в проекте
Эта миграция выполняется при первом запуске приложения и формирует структуру таблиц, которая позволяет хранить результаты парсинга JSON-логов.

## AI-контекст
*   Схема спроектирована с учетом пагинации (индексы на трофеи и уровень) для будущих лидербордов.
*   Используется тип `sa.JSON()` для хранения сложных структур (формы предметов, статы), что обеспечивает гибкость при обновлении игровых механик.
