# [Миграция 0002: индексы и timestamps (0002_profile_indexes_and_timestamps.py)](../../../../../Backend/DB/migrations/versions/0002_profile_indexes_and_timestamps.py)

## Назначение
Вторая миграция: добавляет timestamp-поля профиля, уникальный индекс по `user_id` и триграммные GIN-индексы для быстрого поиска. Делает схему самодостаточной (индексы существуют сразу после `upgrade head`, без ожидания старта приложения).

## Связи (Dependencies)
*   Зависит от [0001](0001_initial_schema.md) (`down_revision='0001'`).
*   Дублирует логику, ранее жившую в [api.py (create_indexes)](../../../playerdata/api.md); проверяется скриптом [verify_indexes.py](../../../../scripts/verify_indexes.md).

## Дедупликация перед уникальным индексом — **КРИТИЧНО**
На «старых» БД мог накопиться дубль профилей одного игрока (баг до upsert). Прямое `CREATE UNIQUE INDEX uix_profile_user_id` на таких данных падает с `UniqueViolation: Key (user_id)=... is duplicated` и роняет старт бэкенда. Поэтому перед созданием индекса вызывается `_dedupe_profiles_by_user_id()`:
*   для каждого непустого `user_id` оставляет профиль с **максимальным `pk`** (самый свежий);
*   у устаревших профилей удаляет связанные `item` и `hero` (FK без `ON DELETE CASCADE`), затем сами профили;
*   `NULL`-профили не трогаются (частичный индекс их допускает).
Идемпотентно: на чистой БД удалять нечего.

## Подробное описание `upgrade()` (диалект-зависимо)
1.  **Timestamps**: `profile.created_at`/`updated_at` с `server_default=func.now()` (бэкфилл существующих строк) — поддержка upsert и TTL-политики хранения.
2.  **Частичный уникальный индекс** по `profile.user_id` (`WHERE user_id IS NOT NULL`) — защита от дублей профиля одного игрока, допуская NULL.
3.  **`pg_trgm` + GIN-индексы** на `itemdefinition(name, item_id)` (только PostgreSQL).
*   `_dialect()` определяет СУБД; на SQLite триграммные индексы пропускаются.

## AI-контекст
*   Индексы вынесены в миграцию намеренно (раньше создавались в рантайме). На SQLite часть шагов — no-op. `func.now()` бэкфиллит таймстампы существующим строкам.
