# [Сброс базы данных (reset_db.py)](../../../Backend/DB/reset_db.py)

## Назначение
`reset_db.py` — это инструмент администратора/разработчика для полной очистки и пересоздания структуры базы данных. Он полезен при отладке, когда схема данных изменилась настолько сильно, что миграции Alembic становятся избыточными.

---

## Логика работы

### 1. Безопасность
Скрипт запрашивает ручное подтверждение. Пользователь должен набрать слово `DELETE`, чтобы продолжить.

### 2. Механизм удаления
Скрипт пробует два способа очистки:
*   **Способ А (SQLModel)**: Импортирует все модели (`Profile`, `Item`, `Hero`) и вызывает `SQLModel.metadata.drop_all`. Это корректно удаляет таблицы, о которых знает приложение.
*   **Способ Б (Raw SQL)**: Если импорт моделей не удался, скрипт выполняет жесткую очистку схемы `public` (только для PostgreSQL):
    ```sql
    DROP SCHEMA public CASCADE;
    CREATE SCHEMA public;
    ```

### 3. Пересоздание
После удаления всех данных скрипт вызывает `SQLModel.metadata.create_all`, создавая пустую структуру таблиц на основе текущих моделей Python.

---

## Применение (Usage)
```bash
python Backend/DB/reset_db.py
```

## AI-контекст
*   **Внимание!** Этот скрипт удаляет ВСЕ данные, включая профили пользователей и накопленную статистику. Никогда не запускайте его на продакшене.
*   После сброса базы через этот скрипт, рекомендуется запустить `alembic stamp head`, чтобы Alembic считал базу актуальной.
