# [Идемпотентный бутстрап БД (bootstrap.py)](../../../Backend/DB/bootstrap.py)

## Назначение
`bootstrap.py` — это критически важный системный скрипт, который гарантирует корректное состояние схемы базы данных перед запуском бэкенда. Он решает проблему "краш-лупа" при развертывании, когда база данных уже существует, но Alembic еще не знает о её состоянии.

---

## Логика "Умного" запуска

Скрипт проверяет состояние таблиц через `sqlalchemy.inspect` и выбирает одну из стратегий:

### Стратегия А: Штамповка (`stamp head`)
*   **Условие**: В БД **нет** таблицы `alembic_version`, но **есть** прикладные таблицы (например, `profile` или `itemdefinition`).
*   **Ситуация**: Это происходит, если база данных была создана вручную (через `SQLModel.metadata.create_all`) до внедрения Alembic.
*   **Действие**: Alembic помечает текущее состояние базы как "Head" (актуальное), не пытаясь создать таблицы заново. Это предотвращает ошибку `Table already exists`.

### Стратегия Б: Обновление (`upgrade head`)
*   **Условие**: База данных пуста ИЛИ уже находится под управлением Alembic.
*   **Действие**: Выполняется стандартное применение всех миграций.

---

## Применение (Usage)
Скрипт запускается в Docker-контейнере или CI/CD пайплайне перед стартом основного сервера:
```bash
python -m Backend.DB.bootstrap
```

## Связи (Dependencies)
*   **Конфигурация**: Загружает [alembic.ini](../../alembic.md).
*   **Движок**: Использует `engine` из [database.py](database.md).
*   **Скрипты миграций**: Взаимодействует с папкой [migrations/env.py](migrations/env.md).

## AI-контекст
*   Скрипт делает деплой **идемпотентным** — его можно запускать сколько угодно раз, и он не приведет к порче данных.
*   Если бэкенд падает при старте с ошибками БД, в первую очередь стоит проверить логи выполнения `bootstrap.py`.
