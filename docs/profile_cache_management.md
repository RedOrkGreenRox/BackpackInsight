# Управление Кэшем Профиля

## Обзор

Система управления кэшем профиля позволяет автоматически очищать данные профиля при переходе на страницу профиля через кнопки/ссылки, но сохранять кэш при переходе из формы загрузки.

## Как это работает

### 1. Автоматическая очистка кэша

При переходе на страницу `/profile` система проверяет:

- **Если есть данные в `history.state`** (nickname, level) → **НЕ очищает** кэш
- **Если нет данных в `history.state`** → **Очищает** весь кэш профиля

### 2. Способы перехода

#### Сохранение кэша:
- Загрузка JSON через форму на главной странице
- Переход с данными профиля

#### Очистка кэша:
- Кнопка "Перейти к профилю (очистить кэш)" на главной странице
- Кнопка "Профиль" в сайдбаре
- Прямой переход по URL `/profile`
- Переход из других разделов (кроме формы)

## Компоненты

### ProfileCacheUtils

Основной класс для управления кэшем:

```typescript
// Полная очистка кэша
ProfileCacheUtils.clearAllProfileCache();

// Проверка наличия кэша
ProfileCacheUtils.hasProfileCache();

// Умная очистка при навигации
ProfileCacheUtils.clearCacheOnNavigation(state);
```

### Gen Router

Интегрирован в роутер для автоматической очистки:

```typescript
// В handleRoute методе
if (cleanPath === '/profile') {
    ProfileCacheUtils.clearCacheOnNavigation(data);
}
```

## Хранилища

### SessionStorage:
- `currentProfileData` - основные данные профиля
- `profileDynamicState` - состояние UI (скролл, сортировка)
- `profileItemsList` - список предметов для навигации

### LocalStorage:
- `profileDynamicState` - резервная копия состояния UI

## UI Элементы

### 1. Кнопка на главной странице
- Текст: "Перейти к профилю (очистить кэш)"
- Стиль: градиентная кнопка с анимацией
- Действие: переход на `/profile` с очисткой кэша

### 2. Кнопка в сайдбаре
- Иконка: профиль
- Текст: "Профиль"
- Tooltip: "Перейти к профилю (очистить кэш)"
- Действие: переход на `/profile` с очисткой кэша

## Логирование

Система логирует действия в консоли:

```
[ProfileCacheUtils] Cache cleared - navigation without profile data
[ProfileCacheUtils] Cache preserved - navigation with profile data
[ProfileCacheUtils] All profile cache cleared
```

## Сценарии Использования

### Сценарий 1: Загрузка нового профиля
1. Пользователь вставляет JSON в форму
2. Нажимает "Посмотреть профиль"
3. Данные передаются через `history.state`
4. **Кэш НЕ очищается**, новые данные загружаются

### Сценарий 2: Очистка и начало заново
1. Пользователь нажимает "Перейти к профилю (очистить кэш)"
2. Переход на `/profile` без данных
3. **Кэш очищается**, страница показывает пустое состояние

### Сценарий 3: Навигация из других разделов
1. Пользователь находится в каталоге предметов
2. Нажимает "Профиль" в сайдбаре
3. **Кэш очищается**, показывается последнее сохраненное состояние или пусто

## Преимущества

1. **Интеллектуальная очистка** - только когда нужно
2. **Сохранение контекста** - при загрузке новых данных
3. **Удобство для пользователя** - явная кнопка очистки
4. **Логирование** - прозрачность работы системы
