# [SSR-страница предмета через index.html ([id].ts)](../../../../Frontend/Web/functions/item/[id].ts)

## Назначение
Cloudflare Function `GET /item/:id`: берёт реальный `index.html`, инжектит в него SEO-мета/JSON-LD для конкретного предмета и отдаёт — так SPA получает правильные превью/мета без отдельного шаблона.

## Связи (Dependencies)
*   Данные: `GET ${BASE_URL}/api/items` (BASE_URL вычисляется из запроса).
*   Шаблон: `GET ${BASE_URL}/index.html`.

## Подробное описание `onRequestGet(context)`
1.  `params.id` (иначе `400`).
2.  Фетчит предметы, ищет по слагу имени (инлайн-нормализация); не найден — `404`.
3.  Скачивает статический `index.html` (при сбое — `500`).
4.  Готовит мета: name, description (из `tooltips`, ≤160), image (`/images/items/webp/{id}.webp`), url, keywords, JSON-LD.
5.  Инжектит мета в HTML и возвращает страницу.

## AI-контекст
*   Отличие от [api/item/[id].ts](../api/item/[id].md): тот генерирует HTML с нуля, этот — **патчит существующий `index.html`** (ближе к реальному SPA). BASE_URL здесь динамический (из хоста запроса), а не захардкожен.
