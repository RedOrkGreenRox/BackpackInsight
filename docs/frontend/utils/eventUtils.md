# Шина событий (`eventUtils.ts`)

## Назначение
Этот файл реализует легковесную шину событий (Event Bus). Она необходима для взаимодействия между независимыми [Страницами (Бранчами)](../ground/roots/Branch.md) без создания жестких зависимостей. Это единственный способ «общения» модулей, которые не знают о существовании друг друга.

## Связи (Dependencies)
*   **Используют этот файл**:
    *   [Обработчик черновиков](../ground/branches/main/_main/managers/draft/DraftEventHandler.md) (для синхронизации ввода).
    *   [Менеджеры профиля](../ground/branches/profile/_profile/managers/ProfileManager.md) (для уведомлений об обновлении данных).
    *   [Система навигации](../ground/roots/_roots/shell/navigation/navigation.md) (событие `pageChange`).

## Подробное описание методов

### `on(event: string, callback: Function)`
*   **Логика**: Добавляет функцию-обработчик в список слушателей конкретного события.
*   **Детали**: Если события еще нет в реестре, создается новый массив. Позволяет нескольким компонентам одновременно реагировать на одно и то же действие (например, очистить кеш и показать уведомление при ошибке).

### `off(event: string, callback: Function)`
*   **Логика**: Удаляет конкретный колбэк из списка слушателей.
*   **Сложность**: Требует передачи той же ссылки на функцию, которая использовалась при подписке. 
*   **Зачем это нужно**: Критически важно для предотвращения утечек памяти и «зомби-событий», когда удаленный с экрана компонент продолжает пытаться обновлять интерфейс.

### `emit(event: string, data?: any)`
*   **Логика**: Запускает выполнение всех подписанных колбэков, передавая им необязательный объект данных.
*   **Детали**: Выполнение происходит синхронно, в порядке очереди подписки. 

## AI-контекст
*   **Архитектурный стиль**: Проект уходит от глубокой вложенности колбэков в пользу этой шины.
*   **Отладка**: Если действие произошло (например, файл загружен), но UI не изменился — проверяйте цепочку `emit` -> `on`.
