# [Frontend/Web/ground/utils/icon-parser.ts](../../../../Frontend/Web/ground/utils/icon-parser.ts)

## Назначение
Этот файл — один из самых сложных и важных в проекте. Он реализует **интеллектуальный парсинг игровых описаний** для **Backpack Brawl**. Его задача: превратить сырой текст с бэкенда (из `items_tooltips.json`) в богатый HTML-интерфейс, где ключевые слова, статы и механики подсвечены, а названия характеристик заменены иконками.

## Связи (Dependencies)
*   **Изображения**: Ссылается на ассеты в `/images/fonticon/`, используя форматы WebP и AVIF.
*   `[global.d.ts](../../types/global.md)`: Используется для типизации (хотя файл в основном работает со строками).
*   `[_vars.scss](../../ground/roots/_roots/_vars.md)`: CSS-классы `value-text` и `text-default`, которые добавляются парсером, определены в этом файле.
*   `[ItemDetailBranch.ts](../../ground/branches/itemDetail/ItemDetailBranch.md)`: Использует парсер для отображения способностей предмета.

## Константы и конфигурация

### Массивы иконок и маппинг
*   `ICON_FILES`: Список имен файлов иконок (например, "Damage", "Poison", "Stamina").
*   `ALIAS_MAP`: Словарь синонимов. Позволяет преобразовывать игровые термины в имена файлов (например, "Melee Weapon" -> "MeleeWeapon", "Coin" -> "Gold").
*   `TYPE_GENERATOR_ALIASES`: Специальные алиасы для типов предметов (например, "Armor" -> "TypeArmor").

### Правила исключений
*   `FORBIDDEN_NAMES`: Список слов, которые начинаются с большой буквы, но **не** являются названиями предметов (например, "Gain", "When", "After"). Это предотвращает ложную подсветку глаголов как имен собственных.
*   `HEADER_EXCEPTIONS...`: Слова, которые не должны считаться заголовками триггеров, даже если они стоят в начале строки или перед двоеточием.

## Вспомогательные функции

### `createIconHtml(iconName, title)`
Генерирует сложную HTML-структуру `<picture>`.
*   Включает несколько тегов `<source>` для поддержки `image/avif` и `image/webp`.
*   Добавляет атрибут `title` для отображения подсказки при наведении.
*   Использует `loading="lazy"` для оптимизации производительности.

### `getAlphaId(num)`
Преобразует порядковый номер в буквенный ID (0 -> A, 1 -> B, 26 -> AA). Используется для создания временных уникальных плейсхолдеров иконок вида `[[ICNA]]`.

### `replaceOutsideSpans(str, regex, handler)` — **КРИТИЧЕСКАЯ ФУНКЦИЯ**
Решает проблему «вложенного парсинга». Она гарантирует, что регулярные выражения не будут изменять текст, который уже находится внутри HTML-тегов (`<span>`, `<a>`, `<picture>`).
*   **Как это работает**: Функция разбивает строку на «безопасные» (чистый текст) и «защищенные» (внутри тегов) участки. Регулярное выражение применяется только к безопасным участкам, предотвращая порчу уже сформированного HTML-кода.

## Основная функция: `parseTextWithIcons(text)`

Проходит через серию этапов (шагов) трансформации текста:

### Шаг 1 и 2: Скрытие иконок (Tokenization)
Парсер ищет все слова из `ALL_TRIGGER_WORDS` в тексте. Найденные слова заменяются на временные токены `[[ICN...]]`. 
*   *Зачем*: Чтобы названия иконок (например, "Hammer") не были случайно изменены правилами подсветки имен предметов на последующих шагах.

### Шаг 3: Обработка заголовков (Triggers)
Ищет начала строк или фразы перед двоеточием, которые являются триггерами (например, "On hit:", "Start of Dawn:").
*   Оборачивает их в `<span class="value-text">`.
*   Проверяет исключения из `HEADER_EXCEPTIONS`, чтобы не подсвечивать обычные слова.

### Шаги 4 и 5: Числа и Проценты
Ищет числовые значения (`+5`, `-10%`, `2.5`).
*   Использует `NUMBER_FOLLOW_TRIGGERS`, чтобы понять, относится ли число к игровой механике (например, "5 seconds", "10% chance").
*   Оборачивает числа в `value-text`.

### Шаг 6: Имена предметов и Ключевые слова
*   Подсвечивает названия предметов (слова с большой буквы, например "Wooden Sword").
*   Исключает слова из `FORBIDDEN_NAMES`.
*   **Специальная логика "Star"**: В **Backpack Brawl** механика "Звездочек" (`Star`) часто ссылается на тип предмета (например, "Star Food"). Парсер находит иконку звезды и подсвечивает следующее за ней слово как тип предмета.

### Шаг 6.2: Специфические слова
*   Окрашивает само слово "item" или "items", если оно стоит после числа или префикса ("common item").

### Финальная сборка
1.  **Возврат иконок**: Заменяет токены `[[ICN...]]` на реальный HTML-код иконок.
2.  **Переносы строк**: Меняет `\n` на `<br>`.
3.  **Обертка обычного текста**: Весь оставшийся текст, который не попал в теги, оборачивается в `<span class="text-default">`.
4.  **Исправление вложенности**: Удаляет дублирующиеся спаны (например, если `text-default` оказался внутри `value-text`).

## Функция `generateIconsOrText(words)`
Упрощенная версия для рендеринга списка тегов или типов предметов (например, в заголовке карточки).
*   Принимает массив строк.
*   Для каждой строки проверяет, есть ли для неё иконка.
*   Если есть — рисует иконку, если нет — выводит текст в спане `text-fallback`.

## AI-контекст
*   **Сложность Regex**: Файл содержит тяжелые регулярные выражения. При их изменении нужно учитывать производительность и избегать "Catastrophic Backtracking".
*   **Синхронность**: Этот парсер должен работать абсолютно так же, как и его Python-копия на бэкенде. Любая рассинхронизация приведет к тому, что в превью и в игре способности будут выглядеть по-разному.
