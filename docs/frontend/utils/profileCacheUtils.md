# [Frontend/Web/utils/profileCacheUtils.ts](../../../Frontend/Web/utils/profileCacheUtils.md)

## Назначение
Утилита для управления историей загруженных профилей на устройстве пользователя. Она позволяет быстро переключаться между своими аккаунтами или профилями друзей без повторной загрузки файлов.

## Связи (Dependencies)
*   `[global.d.ts](../types/global.md)`: Использует интерфейс `Profile` для хранения сокращенных версий данных.
*   `[eventUtils.ts](./eventUtils.md)`: Оповещает систему о том, что список доступных профилей в кеше изменился (например, после удаления).

## Ключевая логика
Использует `LocalStorage` для долгосрочного хранения данных.

### Основные возможности:
*   **Сохранение**: При успешном анализе нового профиля через `[ApiService.ts](./ApiService.md)`, его метаданные (никнейм, уровень, аватар) сохраняются в список истории.
*   **Лимиты**: Управляет максимальным количеством хранимых профилей (например, последние 10), удаляя старые для экономии места.
*   **Быстрый доступ**: Позволяет мгновенно получить данные последнего просмотренного игрока при перезагрузке страницы.

## AI-контекст
*   **Конфиденциальность**: Данные хранятся только локально. ИИ должен учитывать это при проектировании функций "поделиться профилем" — данные в этом кеше недоступны другим пользователям напрямую.
*   **Синхронизация с UI**: Компоненты вроде "Recent Profiles" должны подписываться на события из этого файла.
