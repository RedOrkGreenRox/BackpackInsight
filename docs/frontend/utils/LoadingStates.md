# [Состояния загрузки (LoadingStates.ts)](../../../Frontend/Web/utils/LoadingStates.ts)

## Назначение
Класс управляет визуальным «ожиданием» во всем приложении. Он абстрагирует логику показа спиннеров и скелетонов, позволяя коду бизнес-логики не заботиться о ручном скрытии/показе элементов DOM.

## Связи (Dependencies)
*   **Стили**: Опирается на [Визуальные стили загрузки](../ground/utils/_loading-states/loading-states.md).
*   **Использование**: Внедрен в [Сервис API](./ApiService.md) и [ItemsBranch](../ground/branches/items/ItemsBranch.md).

---

## Подробное описание методов

### `setLoading(id: string, state: boolean)`
*   **Логика**: Находит в DOM элементы с атрибутом `data-loading-id="{id}"` и переключает им CSS-класс `.is-loading`.
*   **Детали**: Позволяет точечно управлять загрузкой (например, заблокировать только кнопку, а не всю страницу).

### `isLoading(id: string): boolean`
*   **Логика**: Возвращает текущий статус процесса по его идентификатору.

### `withLoading<T>(id: string, promise: Promise<T>): Promise<T>`
*   **Сложность**: Самый важный метод-хелпер. 
*   **Процесс**:
    1.  Вызывает `setLoading(id, true)`.
    2.  Запускает переданный `promise`.
    3.  Использует блок `finally`, чтобы гарантированно вызвать `setLoading(id, false)`, даже если запрос завершился с ошибкой.
*   **Зачем**: Предотвращает ситуацию, когда интерфейс остается заблокированным навсегда при сбое сети.

### Скелетоны (Skeletons)
*   `createCardSkeleton(count: number)`: Генерирует HTML-строку с пустыми "мерцающими" карточками предметов.
*   `createProfileSkeleton()`: Создает структуру пустой шапки профиля.

## AI-контекст
При генерации любого асинхронного экшена (например, удаление профиля из истории), всегда оборачивайте его в `withLoading`. Это стандарт "гигиены" UI в **Backpack Insight**.
