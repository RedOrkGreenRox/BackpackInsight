# [Frontend/Web/utils/JsonValidator.ts](../../../Frontend/Web/utils/JsonValidator.ts)

## Назначение
Этот файл выполняет критически важную задачу: первичную валидацию JSON-файла, который пользователь выгрузил из **Backpack Brawl**. Он гарантирует, что данные имеют нужную структуру, прежде чем приложение попытается их отрисовать или отправить на бэкенд.

## Связи (Dependencies)
*   `[global.d.ts](../types/global.md)`: Использует интерфейсы типов для сопоставления структуры входящего JSON с ожидаемыми типами данных.
*   `[eventUtils.ts](./eventUtils.md)`: Генерирует события об ошибках валидации, на которые подписывается интерфейс уведомлений.

## Ключевая логика
Валидатор проверяет наличие обязательных полей, таких как `technical_information` и `game_information` (или их сокращенные версии из игрового лога).

### Основные проверки:
*   **Проверка формата**: Убеждается, что файл вообще является валидным JSON.
*   **Проверка версии**: Сверяет версию данных (поле `AV` или `Actual Version`) на совместимость.
*   **Проверка целостности**: Ищет ключевые узлы: список героев (`Hero`) и список предметов (`Item`). Если их нет, файл считается поврежденным или не относящимся к профилю **Backpack Brawl**.

## AI-контекст
*   **Безопасность**: Это первая линия обороны. Если вы расширяете формат профиля на бэкенде, обязательно добавьте соответствующие правила проверки в этот файл.
*   **Сообщения об ошибках**: Логика формирования текста ошибок должна быть максимально подробной, чтобы ИИ или пользователь понимали, *почему* файл не подошел (например: "Отсутствует секция 'game_information'").
