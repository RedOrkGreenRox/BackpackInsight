# [Frontend/Web/utils/itemCache.ts](../../../Frontend/Web/utils/itemCache.ts)

## Назначение
Этот файл реализует механизм кеширования статических данных о предметах в `SessionStorage` или `LocalStorage`. Это необходимо, чтобы не запрашивать описание одного и того же «Деревянного меча» у сервера каждый раз, когда пользователь открывает разные профили.

## Связи (Dependencies)
*   `[ApiService.ts](./ApiService.md)`: Используется для загрузки данных с сервера, если их нет в кеше (Cache Miss).
*   `[global.d.ts](../types/global.md)`: Описывает структуру хранимых данных (`ItemDefinition`).

## Ключевая логика
Кеш работает по принципу "Key-Value", где ключом является ID предмета.

### Основные механизмы:
*   **Инициализация**: При старте приложения кеш может подгружать базовый набор популярных предметов.
*   **Lazy Loading**: Предметы запрашиваются у `[ApiService.ts](./ApiService.md)` только в момент обращения к ним, если они отсутствуют в памяти.
*   **Инвалидация**: Метод очистки кеша при выходе новой версии игры **Backpack Brawl**, чтобы актуализировать изменившиеся статы предметов.

## AI-контекст
*   **Производительность**: Это ключевой файл для оптимизации чанков и сетевого трафика. При работе со списками предметов (Items Grid), всегда проверяйте, идет ли запрос через этот кеш.
*   **Синхронность**: Помните, что получение данных из кеша — синхронно, а из API — асинхронно. Логика компонентов должна уметь обрабатывать оба сценария.
