# [Frontend/Web/ground/utils/ItemPreviewPrefetchService.ts](../../../Frontend/Web/ground/utils/ItemPreviewPrefetchService.ts)

## Назначение
Сервис для предварительной загрузки (prefetching) данных и изображений предметов. Он позволяет приложению работать мгновенно: когда пользователь только наводит мышь на ссылку предмета, данные уже находятся в памяти, а картинка — в кеше браузера.

## Связи (Dependencies)
*   `[SlugService.ts](./SlugService.md)`: Для генерации уникальных ключей кеша.
*   `[ItemIconService.ts](./ItemIconService.md)`: Для получения путей к изображениям перед их предзагрузкой.

## Подробное описание методов

### `prefetch(item, imageSrc)` (static)
*   Сохраняет объект `ItemDefinition` во внутреннюю карту `previews`.
*   Вызывает `preloadImage` для физической загрузки картинки.
*   Это позволяет открыть детальную страницу предмета без показа «белых пятен» загрузки.

### `get(rawNameOrSlug)` (static)
*   Позволяет быстро извлечь данные предмета из кеша предзагрузки по его имени или слагу.

### `preloadImage(imageSrc)` (static)
*   **Логика**: Создает невидимый объект `new Image()` и устанавливает ему `src`.
*   Использует `img.decoding = 'async'`, чтобы процесс декодирования не блокировал отрисовку основного интерфейса игры.
*   Ведет учет уже загруженных ссылок в `preloadedImages`, чтобы не загружать одно и то же дважды.

## AI-контекст
*   **UX**: Этот сервис — секрет «плавности» интерфейса. При генерации списков предметов всегда вызывайте `prefetch` для элементов, которые находятся в зоне видимости (Viewport).
