# [Frontend/Web/ground/utils/SearchTermService.ts](../../../Frontend/Web/ground/utils/SearchTermService.ts)

## Назначение
Мощный движок для **семантического поиска** по предметам **Backpack Brawl**. Он позволяет находить предметы не только по точному названию, но и по синонимам, похожим терминам или переводам (например, запрос "яд" найдет предметы с "Poison").

## Связи (Dependencies)
*   **Словари**: Загружает `term-aliases.ru.json` — базу связей между игровыми терминами.
*   `[i18n.ts](../ground/localization/i18n.md)`: Используется для определения текущего языка поиска.

## Подробное описание методов и алгоритмов

### `normalizeText(text)` (static)
Критическая функция очистки текста:
*   Разделяет CamelCase (например, "FireSword" -> "Fire Sword").
*   Удаляет спецсимволы, нормализует Unicode (убирает диакритику).
*   Заменяет "ё" на "е" для русского поиска.
*   Приводит всё к нижнему регистру.

### `expandQuery(query)` (static)
Расширяет поисковый запрос пользователя. Если пользователь ввел "dagger", метод добавит к запросу все связанные термины ("кинжал", "острие"), найденные в индексе.

### `getWeightedExpandedQueryTerms(query)` (static)
Возвращает список терминов с их «весом» (релевантностью).
*   **Оригинальные слова** имеют вес `0`.
*   **Прямые синонимы** получают вес от `0.08` до `0.18`.
*   Сортирует выдачу так, чтобы наиболее точные совпадения были выше.

### `semanticRank(text, query)` (static)
Вычисляет "оценку близости" текста предмета к запросу.
*   `0`: Прямое совпадение токена.
*   `1`: Совпадение через расширенный синоним.
*   `2`: Нет совпадений.

### `indexWeightedTerms` (private static)
Строит граф зависимостей между словами при старте приложения. Это позволяет мгновенно находить связи "многие-ко-многим" между терминами.

## AI-контекст
*   **Сложность**: Это самый ресурсозатратный сервис на фронтенде. Он строит инвертированный индекс в памяти.
*   **Токенизация**: Поиск работает по словам (токенам). Минимальная длина слова для расширения — 4 символа (`MIN_PREFIX_LENGTH`).
*   **Fuzzy Search**: В отличие от Fuse.js, этот сервис работает на основе **знаний** (синонимов), а не простого расстояния Левенштейна, что делает поиск в разы качественнее для игроков.
