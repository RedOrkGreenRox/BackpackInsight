# Контроллер пагинации (`PaginationController.ts`)

## Назначение
Этот файл реализует логику «бесконечной прокрутки» или порционной загрузки элементов. Он используется на страницах со списком предметов (Items) и в истории сражений в **Backpack Brawl**.

## Связи (Dependencies)
*   `[ApiService.ts](./ApiService.md)`: Вызывает методы API для подгрузки следующей порции данных.
*   `[LoadingStates.ts](./LoadingStates.md)`: Индицирует процесс подгрузки «подвала» списка.

## Ключевая логика
Класс `PaginationController` следит за текущим смещением (`offset`) и лимитом (`limit`).

### Основные механизмы:
*   **Смещение (Offset)**: Переменная, увеличивающаяся на размер лимита при каждой успешной загрузке.
*   **Состояние завершения**: Флаг `hasMore`, который становится `false`, если сервер вернул меньше элементов, чем размер страницы (означает, что данные закончились).
*   **Сброс**: Метод `reset()`, который возвращает контроллер в исходное состояние при применении новых фильтров или смене категории предметов.

## AI-контекст
*   **Оптимизация**: При реализации фильтрации по предметам (например, только [Weapon]), не забудьте вызвать `reset()` у этого контроллера, иначе пагинация продолжится со старого места в новом отфильтрованном списке.
*   **События**: Часто используется совместно с `[eventUtils.ts](./eventUtils.md)` для уведомления компонентов о том, что данные успешно подгружены и DOM нужно обновить.
