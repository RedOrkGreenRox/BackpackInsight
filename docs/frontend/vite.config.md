# [Конфигурация Vite (vite.config.ts)](../../Frontend/Web/vite.config.ts)

## Назначение
`vite.config.ts` — основной файл сборки фронтенда. Он настраивает процесс компиляции TypeScript, обработки стилей и оптимизации ресурсов для BackpackInsight.

---

## Ключевые возможности

### 1. Алиасы путей
Настроены короткие пути для упрощения импортов:
*   `@roots` -> `ground/roots` (ядро)
*   `@branches` -> `ground/branches` (страницы)
*   `@utils` -> `ground/utils` (сервисы)

### 2. Оптимизация Чанков (`manualChunks`)
Реализована сложная стратегия разбиения кода для ускорения первой загрузки:
*   **Vendor-чанки**: Тяжелые библиотеки (`html2canvas`, `fuse.js`, `aos`) выделены в отдельные файлы.
*   **Page-чанки**: Каждая страница (Main, Profile, Items) собирается в свой изолированный файл и загружается только при переходе.
*   **App-shared**: Ядро приложения (i18n, ApiService, LoadingStates) объединено в общий чанк для эффективного кэширования.

### 3. Обработка ресурсов
*   **Единый CSS**: `cssCodeSplit: false` — стили всех страниц собираются в один файл. Это предотвращает «мигание» интерфейса (FOUC) при быстрой навигации в SPA.
*   **Версионирование**: К именам всех сгенерированных файлов добавляется хеш контента, что позволяет использовать бесконечное кэширование (`immutable`).

---

## AI-контекст
*   Проект использует `Vite 8` и требует `Node.js >= 20.19`.
*   Все статические файлы из папки `/static` доступны в коде по прямым путям.
