# [Тесты логики фронтенда (tests/*.test.ts)](../../../Frontend/Web/tests/)

## Назначение
Директория `tests/` содержит набор unit-тестов для проверки критически важной бизнес-логики клиентской части приложения. Тесты написаны с использованием фреймворка **Vitest**.

---

## Проверяемые модули

### 1. `json_validator.test.ts`
Проверяет корректность работы встроенного валидатора JSON-логов:
*   Распознавание синтаксических ошибок Unity-логов.
*   Точное определение строки и колонки ошибки.
*   Безопасное экранирование HTML-тегов при подсветке ошибок.

### 2. `utils.test.ts`
Тестирует базовые вспомогательные функции:
*   `SlugService` — корректность генерации URL-адресов из имен предметов.
*   `ImageFormatService` — правильный выбор формата (AVIF/WebP) в зависимости от браузера.
*   `MetaService` — динамическая подмена SEO-тегов.

### 3. `sort_controller.test.ts`
Проверяет алгоритмы сортировки инвентаря игрока:
*   Сортировка по уровню, редкости и имени.
*   Сохранение порядка (stable sort) при совпадении параметров.

### 4. `items_logic.test.ts` / `profile_logic.test.ts`
Интеграционные тесты для менеджеров страниц:
*   Загрузка данных из кэша.
*   Расчет суммарных статов профиля на клиенте.
*   Фильтрация коллекции предметов.

---

## AI-контекст
*   Тесты используют `jsdom` для имитации работы в браузере и `vi.mock` для изоляции от внешних API и локализации.
*   Запуск тестов осуществляется командой `npm test` из папки `Frontend/Web`.
