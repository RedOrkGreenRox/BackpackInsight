# [Frontend/Web/ground/utils/ItemIconService.ts](../../../../Frontend/Web/ground/utils/ItemIconService.ts)

## Назначение
Централизованный сервис для сопоставления данных предмета с именем его файла-изображения. В **Backpack Brawl** не всегда имя предмета напрямую совпадает с именем файла (из-за спецсимволов или игровых механик «скрытых» предметов).

## Связи (Dependencies)
*   `[SlugService.ts](SlugService.md)`: Используется для очистки имен и преобразования их в безопасные URL-адреса.
*   `[global.d.ts](../types/global.md)`: Использует интерфейс `ItemDefinition` для доступа к характеристикам предмета.
*   **Изображения**: Результат (слаг) указывает на файл в [папке предметов (/images/items/)](../../static/images/items/index.md).

## Подробное описание логики

### `getImagePath(item: ItemDefinition): string` (static)
Главный метод разрешения путей. Работает по цепочке приоритетов:

1.  **Маскировка (Masked Items)**: Сначала проверяет словарь `MASKED_ITEMS`. Некоторые предметы в игре имеют разные названия, но одинаковые иконки (например, 'Suspicious Sausage' -> 'tender-sausage').
2.  **Специальные предметы (Special Rarity)**: 
    *   Если предмет имеет редкость 'Special' (например, чертежи ограбления), метод анализирует первый `tooltip`.
    *   Использует регулярное выражение `STEP_PATTERN` для поиска текста "Step I", "Step 2" и т.д.
    *   Если находит римские цифры — конвертирует их через `[SlugService.md](SlugService.md)`.
    *   **Пример**: Предмет "Step III" превратится в файл `heist-plan-3`.
3.  **Обычный Slug**: Если исключений нет, вызывается `SlugService.toSlug()` от названия предмета.

## AI-контекст
*   **Именование ассетов**: Если вы добавляете новые изображения в `/static/images/items/`, убедитесь, что их имена соответствуют результату работы этого сервиса.
*   **Чертежи**: Логика "Heist Plans" (шаги ограбления) — самая хрупкая часть. При изменении формата тултипов в игре, нужно обновить `STEP_PATTERN` здесь.
