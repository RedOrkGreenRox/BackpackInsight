# [Семантический поиск (SearchTermService.ts)](../../../../Frontend/Web/ground/utils/SearchTermService.ts)

## Назначение
Класс `SearchTermService` реализует сложную логику интеллектуального поиска. Он позволяет находить предметы не только по прямому вхождению, но и по синонимам, сокращениям и семантически близким терминам.

---

## Ключевая логика

### 1. Нормализация и Токенизация
Метод `normalizeText` подготавливает текст к анализу:
*   Разделяет CamelCase (напр. `staminaRegen` -> `stamina regen`).
*   Удаляет пунктуацию и спецсимволы.
*   Удаляет диакритические знаки (акценты).
*   Заменяет `ё` на `е`.

### 2. Расширение запроса (Query Expansion)
Используя словарь алиасов (`term-aliases.ru.json`), сервис расширяет поисковой запрос:
*   Если пользователь ищет "хп", сервис добавит в поиск "здоровье", "health", "hp".
*   Каждому термину присваивается вес (0 — оригинал, 0.16 — синоним). Это позволяет [**ItemsBranch**](../../ground/branches/items/ItemsBranch.md) ранжировать результаты.

### 3. Семантическое ранжирование (`semanticRank`)
Алгоритм классифицирует совпадения по трем уровням:
*   **Rank 0**: Прямое вхождение токена в текст.
*   **Rank 1**: Вхождение одного из синонимов.
*   **Rank 2**: Совпадение не найдено.

---

## AI-контекст
*   Сервис строит внутренний индекс `tokenToExpansion` для O(1) поиска связанных терминов.
*   Поддерживает нечеткое сравнение префиксов (минимум 4 символа), что позволяет находить "poisonous" при поиске "poison".
