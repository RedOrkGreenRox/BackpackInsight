# [Frontend/Web/ground/utils/ImageFormatService.ts](../../../../Frontend/Web/ground/utils/ImageFormatService.ts)

## Назначение
Этот сервис отвечает за глобальное определение оптимального формата изображений (AVIF или WebP) при запуске приложения. Вместо того чтобы использовать тяжелые HTML-конструкции `<picture>` для каждого из сотен предметов в рюкзаке, приложение один раз проверяет возможности браузера и в дальнейшем использует прямые пути к изображениям нужного формата.

## Связи (Dependencies)
*   **Изображения**: Ссылается на структуру папок [иконок предметов (/images/items/)](../../static/images/items/index.md) (`avif/` и `webp/`), а также отдаёт путь к [заглушке (/images/placeholder/)](../../static/images/placeholder/index.md) через `placeholderSrc()`. Это центральный сервис выбора формата для всего [дерева изображений](../../static/images/index.md).
*   `[core.ts](../core.md)`: Вызывает метод `init()` при старте приложения.

## Подробное описание методов

### `init(): Promise<SupportedImageFormat>` (static)
*   **Логика**: Выполняет проверку поддержки форматов через метод `supports`.
*   **Приоритет**: Сначала проверяет AVIF (как наиболее эффективный), затем WebP. Если оба не поддерживаются (что маловероятно для современных браузеров), откатывается на WebP.
*   **Результат**: Сохраняет выбранный формат в статическую переменную и добавляет атрибут `data-image-format` к `<html>`, что позволяет использовать формат даже в CSS.

### `itemSrc(imageName: string): string` (static)
*   Генерирует прямой путь к иконке предмета. 
*   **Пример**: Если выбран AVIF, вызов `itemSrc('wooden-sword')` вернет `/images/items/avif/wooden-sword.avif`.

### `placeholderSrc(): string` (static)
*   Возвращает путь к заглушке (placeholder) в выбранном формате. Используется, если основное изображение еще не загрузилось.

### `supports(format): Promise<boolean>` (private static)
*   **Критическая логика**: Пытается загрузить "крошечную" (1x1 пиксель) картинку-тест в формате Base64. 
*   Если изображение загрузилось и имеет размеры > 0 — формат поддерживается.
*   Включен таймаут 800мс на случай зависания декодера.

## AI-контекст
*   **Оптимизация**: Этот сервис — ключевой элемент производительности. Использование одного формата вместо `<picture>` значительно уменьшает количество DOM-узлов при отрисовке больших сеток предметов.
*   **Добавление форматов**: Если в будущем добавится поддержка JPEG XL, её нужно внедрять именно сюда.
