# [Сервис API (ApiService.ts)](../../../../Frontend/Web/ground/utils/ApiService.ts)

## Назначение
`ApiService` — это централизованный шлюз для взаимодействия фронтенда с бэкенд-сервисом. Он инкапсулирует логику HTTP-запросов, обработку ошибок и механизмы повторных попыток (Retry).

---

## Ключевая логика

### 1. Обработка ошибок (`getErrorInfo`)
Сервис классифицирует ошибки на основе HTTP-статусов:
*   **400/413**: Ошибки валидации (некорректный JSON или слишком большой файл). Не подлежат повтору.
*   **429**: Rate limit (превышен лимит запросов). Подлежит повтору.
*   **503**: Бэкенд временно недоступен. Подлежит повтору.

### 2. Экспоненциальный бэкофф (`retryWithBackoff`)
Если запрос завершился сетевой ошибкой или ошибкой сервера, сервис делает до **3 попыток** с увеличивающейся задержкой (1с -> 2с -> 4с). Это значительно повышает стабильность приложения при нестабильном интернете.

### 3. Методы данных
*   **`getProfile(jsonData)`**: Отправляет сырые данные игрока для анализа.
*   **`getItems()`**: Загружает глобальный справочник предметов.

---

## AI-контекст
*   Сервис использует `/api/` прокси, который настроен в [server.ts](../server.md).
*   Логика ретраев особенно важна для Cloudflare Pages, так как холодный старт бэкенд-контейнера может занимать несколько секунд.
