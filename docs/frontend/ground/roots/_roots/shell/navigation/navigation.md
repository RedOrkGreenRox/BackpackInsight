# [Frontend/Web/ground/roots/_roots/shell/navigation/navigation.ts](../../../../../../../Frontend/Web/ground/branches/404/_404/navigation/navigation.ts)

## Назначение
Сердце клиентского роутинга (Routing). Этот файл отвечает за бесшовное переключение между «бранчами» (страницами) без перезагрузки всей страницы.

## Связи (Dependencies)
*   `[Branch.ts](../../../Branch.md)`: Оперирует объектами, наследуемыми от базового класса бранча.
*   `[MainBranch.ts](../../../../branches/main/MainBranch.md)` / `[ProfileBranch.ts](../../../../branches/profile/ProfileBranch.md)`: Импортирует классы конкретных страниц для их инстанцирования.
*   `[eventUtils.ts](../../../../../utils/eventUtils.md)`: Генерирует события `pageChange`, на которые могут реагировать другие части системы (например, скрытие меню).

## Ключевая логика
Использует History API для изменения URL и управления кнопками "Назад/Вперед". 

### Процесс смены страницы:
1.  **Выгрузка**: Вызывает `destroy()` у текущего бранча.
2.  **Загрузка**: Создает экземпляр нового бранча.
3.  **Инициализация**: Вызывает `init()` и `render()` у нового бранча.
4.  **Анимация**: Управляет плавным переходом (fade-in/fade-out).

## AI-контекст
*   **Динамический импорт**: Для оптимизации (пункт «Пересмотр чанков») здесь стоит реализовать динамическую загрузку бранчей, чтобы код страницы профиля не грузился, пока пользователь на главной.
*   **Состояние**: Навигация должна корректно передавать параметры из URL (например, никнейм игрока) в `init()` бранча.
