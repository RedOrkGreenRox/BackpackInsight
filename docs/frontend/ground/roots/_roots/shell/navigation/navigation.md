# [Навигация Shell (navigation.ts)](../../../../../../../Frontend/Web/ground/roots/_roots/shell/navigation/navigation.ts)

## Назначение
Небольшой вспомогательный скрипт «оболочки», отвечающий за **анимированный переход по обычным ссылкам** (полная навигация с перезагрузкой) и за корректное восстановление состояния страницы после кнопки «Назад». Это НЕ SPA-роутер — клиентский роутинг живёт в [Генераторе UI (Gen.ts)](../../../Gen.md).

## Связи (Dependencies)
*   `[global.d.ts](../../../../types/global.md)`: подключается через `/// <reference>` для типов глобальных функций.
*   **DOM/Sidebar**: работает с `document.body` (классы `leaving`, `loaded`, `sidebar-open`) и элементом `#sidebar`.
*   **History/BFCache**: использует событие `pageshow` и флаг `persisted` для обработки back-forward cache.

## Подробное описание (реальная реализация)

### `globalThis.goTo(url: string)`
*   Определяет мобильное устройство (`innerWidth <= 768`) и подбирает задержку анимации (100мс на мобильных, 250мс на десктопе).
*   Добавляет `body.leaving` (запуск анимации ухода), закрывает сайдбар (снимает класс `open`).
*   По таймеру выполняет фактический переход через `location.href = url`.

### Слушатель `pageshow`
*   Если страница восстановлена из BFCache (`event.persisted === true`), снимает классы `leaving`/`sidebar-open` и возвращает `loaded`, чтобы интерфейс не «завис» в состоянии анимации ухода.

## AI-контекст
*   **Не путать с роутером**: здесь именно «жёсткая» навигация по `location.href`. SPA-переходы без перезагрузки, динамические импорты страниц и передача параметров URL — задача `Gen.ts`.
*   **BFCache**: правка анимаций перехода обязательно должна учитывать ветку `pageshow/persisted`, иначе после «Назад» страница останется затемнённой.
