# [Генератор интерфейса и Роутер (Gen.ts)](../../../../Frontend/Web/ground/roots/Gen.ts)

## Назначение
Класс `Gen` — это сердце Vanilla TS движка BackpackInsight. Он реализует логику **Single Page Application (SPA)**, отвечая за навигацию, смену страниц и сохранение их состояния.

---

## Ключевая логика

### 1. Умный Роутинг
Поддерживает как статические пути (`/`), так и динамические параметры (например, `/item/:name`). Алгоритм сопоставления путей (`findRoute`) умеет извлекать параметры из URL и передавать их в методы жизненного цикла страниц.

### 2. Жизненный цикл "Бранчей"
`Gen` управляет переключением страниц по следующему циклу:
1.  **Загрузка**: Подгружает JS-чанк страницы через `import()`.
2.  **Unmount**: Вызывает метод `destroy()` у старой страницы.
3.  **Mount**: Отрисовывает HTML новой страницы и вызывает `init()`.
4.  **Анимация**: Обеспечивает плавный переход через CSS-класс `fade-out`.

### 3. Сохранение состояния (State)
При навигации `Gen` сохраняет текущую позицию скролла в `history.state`. Это позволяет пользователю вернуться на ту же высоту страницы при нажатии кнопки "Назад" в браузере.

### 4. Предзагрузка (Prefetch)
Реализована агрессивная стратегия предзагрузки: как только пользователь наводит курсор на ссылку (`pointerover`), соответствующий модуль страницы начинает загружаться в фоновом режиме.

---

## AI-контекст
*   **Race Condition protection**: Используется `navigationId` для предотвращения ошибок при слишком быстром переключении вкладок. Если пользователь кликнул на страницу Б, пока грузилась страница А, будет отрисована только Б.
*   **Meta Update**: Роутер автоматически вызывает [MetaService](../../utils/MetaService.md) при каждом переходе, обновляя заголовок и описание страницы для SEO.
