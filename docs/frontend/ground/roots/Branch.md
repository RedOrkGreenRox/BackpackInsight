# [Базовый класс страницы (Branch.ts)](../../../../Frontend/Web/ground/roots/Branch.ts)

## Назначение
`Branch` — это абстрактный базовый класс для всех страниц приложения. Он описывает стандартный жизненный цикл и структуру любого раздела сайта.

---

## Методы жизненного цикла

### 1. `getHtml(data)` (абстрактный)
Должен возвращать строку с HTML-разметкой страницы. Вызывается роутером перед вставкой в DOM.

### 2. `init(data)` (абстрактный)
Вызывается **после** того, как HTML был добавлен на страницу. Здесь должна находиться логика навешивания событий, инициализации менеджеров и запросов к API.

### 3. `destroy()` (абстрактный)
Вызывается перед переходом на другую страницу. Здесь необходимо удалять слушатели событий и очищать таймеры, чтобы избежать утечек памяти.

### 4. `getMeta(data)`
Позволяет каждой странице задать уникальный `<title>` и `<meta name="description">` для поисковых систем.

---

## AI-контекст
*   Проект не использует Virtual DOM (как React), поэтому методы `mount` и `unmount` работают напрямую с `innerHTML`. Это обеспечивает максимальную производительность, но требует аккуратности в методе `destroy`.
