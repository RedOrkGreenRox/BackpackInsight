# [Стили 404 (404.scss)](../../../../../Frontend/Web/ground/branches/404/404.scss)

## Назначение
Корневой («бочка») SCSS-файл страницы 404. Сам правил почти не содержит — он **собирает** страницу из атомарных частей через `@use` в строго заданном порядке слоёв (фон → оверлей → тело → контейнер → контент → кнопка).

## Связи (Dependencies)
*   `@use "../../roots/_roots"` — глобальные переменные дизайна и сбросы ([корневые стили](../../roots/_roots/_vars.md)).
*   Подключает атомы: [background](_404/background/background.md), `overlay`, [body](_404/body/body.md), `container`, `title`, `text`, [button](_404/button/button.md).

## Ключевая логика и стили
*   Порядок `@use` важен: задаёт каскад слоёв (z-index/перекрытие) — фон лежит ниже, кнопка собирается последней как финальный интерактивный слой.
*   Единственное собственное правило:
    *   `body.error-404 { @extend .body-404; }` — навешивает базовые стили тела страницы на `<body>` только когда у него класс `error-404`, не затрагивая остальные страницы.

## AI-контекст
*   Это файл-агрегатор. Новые визуальные части 404 добавляются как отдельные атомы и подключаются здесь через `@use`, а не правятся инлайн.
