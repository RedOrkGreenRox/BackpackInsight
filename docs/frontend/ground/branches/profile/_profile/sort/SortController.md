# [Frontend/Web/ground/branches/profile/_profile/sort/SortController.ts](../../../../../../../Frontend/Web/ground/branches/profile/_profile/sort/SortController.ts)

## Назначение
Этот класс управляет интерактивной сортировкой карточек героев в профиле. Он связывает кнопки интерфейса ("По уровню", "По рейтингу", "Инверсия") с DOM-манипуляциями сетки героев.

## Связи (Dependencies)
*   `[i18n.ts](../../../../localization/i18n.md)`: Для перевода названий режимов сортировки в UI.
*   **Изображения**: Ссылается на ассеты в `/images/profile/` для смены иконок кнопок.
*   **Внешние библиотеки**: Использует `AOS` (Animate On Scroll) для обновления анимаций после перестроения сетки.

---

## Подробное описание функций и методов

### Конструктор
*   `constructor(container)`: Принимает контейнер бранча и запускает метод `init()`.

### Приватные методы манипуляции UI
*   `addListener(element, event, handler)`: Вспомогательная функция, которая регистрирует слушатель и сохраняет функцию отписки в массив `cleanupFns`.
*   `updatePicture(img, iconName)`:
    *   **Логика**: Плавно (через прозрачность) меняет иконку кнопки.
    *   **Сложность**: Она автоматически обновляет не только `src` у `<img>`, но и находит все `<source>` внутри родительского `<picture>`, подставляя правильные пути для AVIF и WebP форматов.

### Приватные методы логики сортировки
*   `init()`:
    *   Находит кнопки `#sortToggle`, `#invertToggle` и элементы сетки.
    *   Настраивает переключение режимов: 'level' <-> 'rating'.
    *   Слушает клики и инициирует перерисовку.
*   `getCardValue(card)` / `getCardTiebreaker(card)`:
    *   Извлекает данные из `dataset` атрибутов HTML-элемента карточки.
    *   Для уровня учитывает флаг `prestige`: если он `true`, к значению добавляется виртуальный бонус (+20), чтобы престижные герои были выше обычных того же уровня.
*   `compareCards(a, b)`:
    *   Основная функция сравнения (Comparator).
    *   Реализует каскадную проверку: Сначала по основному значению -> затем по "Tiebreaker" (второстепенному) -> в конце по имени (алфавиту).
*   `applySort(grid)`:
    *   Превращает `HTMLCollection` в массив.
    *   Сортирует его.
    *   Использует `DocumentFragment` для эффективной вставки элементов обратно в DOM за одну операцию отрисовки.

### Публичные методы управления
*   `applySortWithParams(sortBy, inverted)`: Позволяет программно установить режим сортировки (например, при восстановлении состояния страницы из кеша).
*   `destroy()`: Очищает все слушатели событий.

## AI-контекст
*   **Производительность**: Сортировка происходит на клиенте через манипуляцию живыми DOM-узлами. Метод `applySort` использует `requestAnimationFrame`, чтобы избежать "фризов" интерфейса.
*   **Анимации**: После каждого перестроения сетки вызывается `AOS.refresh()`, чтобы эффекты появления карточек сработали заново.
