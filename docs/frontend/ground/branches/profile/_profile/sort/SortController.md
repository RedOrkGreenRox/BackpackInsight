# [Контроллер сортировки (SortController.ts)](../../../../../../../Frontend/Web/ground/branches/profile/_profile/sort/SortController.ts)

## Назначение
Класс управляет интерактивной сортировкой карточек героев в профиле игрока. Он объединяет логику UI (кнопки, иконки) и DOM-манипуляцию для мгновенного изменения порядка элементов без перезагрузки.

---

## Ключевая логика

### 1. Управление UI
Контроллер слушает клики на кнопках `#sortToggle` (тип сортировки) и `#invertToggle` (направление). При каждом клике:
*   Плавно обновляет иконки через метод `updatePicture` (с эффектом прозрачности 200 мс).
*   Обновляет текстовые метки через систему локализации.

### 2. Сортировка DOM (`applySort`)
Использует прямой доступ к дочерним элементам сеток (`#mainHeroesGrid` и `#statsHeroesGrid`):
1.  Извлекает значения из `data-` атрибутов HTML-элементов (level, rating, prestige).
2.  Вычисляет приоритет: уровень престижа (+20 к уровню) учитывается в расчетах.
3.  Переупорядочивает элементы с использованием `DocumentFragment` для минимизации перерисовок (Reflow).

### 3. Алгоритм сравнения
*   **Основное поле**: Уровень или Рейтинг.
*   **Tiebreaker**: Если основные значения равны, используется второе поле для разрешения конфликта.
*   **Алфавит**: Финальный уровень при полном совпадении цифр.

---

## AI-контекст
*   Метод `applySortWithParams` позволяет внешним менеджерам (например, восстановителю состояния) программно задавать порядок элементов.
*   После каждой пересортировки вызывается `AOS.refresh()`, чтобы анимации появления корректно срабатывали для элементов на новых позициях.
