# [Рендерер секции героев (HeroesSectionRenderer)](../../../../../../Frontend/Web/ground/branches/profile/_profile/heroes/heroes-section.ts)

## Назначение
Этот класс формирует контейнер для списка героев в профиле. Он включает в себя заголовок секции, элементы управления сортировкой и саму сетку с карточками героев.

## Связи (Dependencies)
*   **Локализация**: Использует [Систему i18n](../../../../localization/i18n.md) для формирования заголовка "Герои (X)" и подписей кнопок.
*   **Карточки**: Вызывает [HeroCardRenderer](./hero-card.md) для отрисовки каждого героя.
*   **События**: Генерирует кнопки `#sortToggle` и `#invertToggle`, которые обрабатываются [Контроллером сортировки](../sort/SortController.md).

---

## Подробное описание методов

### `static render(data: ProfileData)`
*   **Первичная сортировка**: В отличие от других рендереров, этот метод выполняет сортировку массива героев прямо в момент генерации HTML. Это гарантирует, что даже до инициализации JS-контроллеров герои будут стоять в правильном порядке (Уровень -> Рейтинг -> Имя).
*   **Панель управления**: Отрисовывает кнопки смены режима сортировки и инверсии.
*   **Сетка**: Оборачивает массив отрендеренных карточек в контейнер `.main-heroes-grid` с ID `#mainHeroesGrid`.

### `private static getHeroesTitle(count)`
Хелпер для получения локализованной строки заголовка с числом героев.

## AI-контекст
*   **SEO и Доступность**: Заголовок секции выводится тегом `<h2>`, обеспечивая правильную иерархию документа.
*   **Синхронизация**: Иконки внутри кнопок сортировки имеют ID (`#sortIcon`, `#invertIcon`), так как их `src` будет динамически меняться в [Контроллере сортировки](../sort/SortController.md) без перерисовки всей секции.
