# [Рендерер карточки героя (HeroCardRenderer)](../../../../../../../Frontend/Web/ground/branches/profile/_profile/heroes/hero-card.ts)

## Назначение
Этот класс отвечает за генерацию HTML-кода для больших карточек героев, которые отображаются в основной сетке профиля. Карточка содержит детальную информацию об опыте героя, его уровне, лиге и предоставляет интерфейс для смены скинов.

## Связи (Dependencies)
*   **Типы**: Использует интерфейс [Герой](../utils/profile-types.md).
*   **Управление**: Генерирует кнопки `.skin-btn`, события на которых слушает [Оркестратор профиля](../managers/ProfileManager.md).
*   **Изображения**: Портреты/скины берёт из [папки героев (/images/heroes/)](../../../../../static/images/heroes/index.md); смену скина оркеструет [ProfileSkinsManager](../managers/ProfileSkinsManager.md).
*   **Стили**: Опирается на анимации из `AOS` (атрибут `data-aos="fade-up"`).

---

## Подробное описание методов

### `static render(hero: Hero)`
Создает структуру карточки.
*   **Dataset-атрибуты**: Прописывает `data-level`, `data-rating`, `data-prestige` и `data-hero-name`. Это критично для [Контроллера сортировки](../sort/SortController.md), который работает напрямую с этими значениями в DOM.
*   **Смена скинов**: Добавляет пустые кнопки `.prev-skin` и `.next-skin`. Логика переключения образов реализована в [Менеджере скинов](../managers/ProfileSkinsManager.md).
*   **Изображение**: Генерирует сложный путь к скину героя. По умолчанию используется скин из данных профиля (`hero.skin_num`).
*   **Лига и Опыт**: Отображает текущий прогресс до следующего уровня и иконку ранга.

### `private static formatRating(rating)`
Аналогично логике в [Рендерере заголовка](../header/header.md), вычисляет отображаемое число очков внутри лиги (сброс каждые 500 очков до 5000).

## AI-контекст
*   **Интерактив**: Карточка является чисто визуальным компонентом. Вся логика "жизни" (клики, свайпы скинов) навешивается позже в [Оркестраторе](../managers/ProfileManager.md).
*   **Анимация**: Плавная смена скинов (`opacity 0.2s`) заложена в инлайновых стилях изображения для обеспечения мгновенного отклика без ожидания загрузки CSS.
