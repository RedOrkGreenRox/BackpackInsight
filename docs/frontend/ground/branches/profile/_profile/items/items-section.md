# [Рендерер секции предметов (ItemsSectionRenderer)](../../../../../../Frontend/Web/ground/branches/profile/_profile/items/items-section.ts)

## Назначение
Этот класс отвечает за отрисовку всей коллекции предметов игрока. Он создает контейнер сетки (grid) и панель управления переключением режимов сортировки.

## Связи (Dependencies)
*   **Карточки**: Вызывает [ItemCardRenderer](./item-card.md) для каждой единицы в массиве `data.items`.
*   **Локализация**: Использует [Систему i18n](../../../../localization/i18n.md) для текстов кнопок и заголовков.
*   **Состояние**: Принимает `currentItemSort` из [Оркестратора профиля](../managers/ProfileManager.md), чтобы отобразить правильную подпись на кнопке.

---

## Подробное описание методов

### `static render(data, currentItemSort)`
*   Отрисовывает заголовок "Предметы (X)".
*   Генерирует кнопку `#itemSortToggle`. В отличие от героев, здесь используется простая кнопка-переключатель текста.
*   **Массив в строку**: Итерирует список предметов и вызывает рендер карточки. 
*   **Оптимизация**: Контейнер имеет ID `#profileItemsGrid`, что позволяет [Оркестратору](../managers/ProfileManager.md) быстро заменять его содержимое при смене сортировки без перерисовки всей секции.

### Приватные хелперы локализации
*   `getItemsTitle(count)`
*   `getItemsSortRarity()`
*   `getItemsSortLevel()`
Эти методы инкапсулируют вызовы `t()` для соблюдения «гигиены» кода и упрощения чтения HTML-шаблона.

## AI-контекст
*   **Верстка**: Секция использует класс `.items-grid`, который настраивает адаптивную сетку (flex или grid в зависимости от устройства).
*   **Производительность**: При большом количестве предметов (100+) этот метод генерирует длинную строку. Рекомендуется использовать виртуализацию, если коллекция вырастет сверх разумных пределов.
