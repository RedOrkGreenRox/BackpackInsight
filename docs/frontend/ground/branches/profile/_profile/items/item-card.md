# [Рендерер карточки предмета (ItemCardRenderer)](../../../../../../Frontend/Web/ground/branches/profile/_profile/items/item-card.ts)

## Назначение
Генерирует HTML-код для отдельной карточки предмета в коллекции игрока. Карточка является кликабельной ссылкой, ведущей на страницу детальной информации о предмете.

## Связи (Dependencies)
*   **Типы**: Использует интерфейс [Предмет](../utils/profile-types.md).
*   **Изображения**: Ссылается на [Сервис форматов изображений](../../../../../utils/ImageFormatService.md) для получения путей к картинкам.
*   **Навигация**: Использует [SlugService](../../../../../utils/SlugService.md) для формирования URL ссылки.

---

## Подробное описание методов

### `static render(item: Item, index: number)`
Создает визуальную плитку предмета.
*   **Cards Info**: Если у предмета есть лимит карточек (`cards_need !== -1`), выводит текущий прогресс (например, "(5 / 10)").
*   **Динамическая ссылка**: Оборачивает карточку в тег `<a>` с атрибутом `data-link`. Это позволяет [Навигации Shell](../../../../roots/_roots/shell/navigation/navigation.md) перехватить клик и выполнить переход без перезагрузки страницы.
*   **Оптимизация рендеринга**:
    *   Применяет `decoding="async"` и `fetchpriority="low"`, чтобы загрузка сотен иконок не мешала отрисовке основного интерфейса.
    *   Устанавливает `data-aos-delay` на основе индекса предмета. Это создает эффект "волны" при появлении списка предметов.
*   **Стилизация**: Добавляет CSS-класс `rarity-{rarity}`, что позволяет окрашивать название редкости в цвета из [глобальных переменных](../../../../roots/_roots/_vars.md).

## AI-контекст
*   **Fallback**: Атрибут `data-fallback` на картинке является меткой для [Оркестратора профиля](../managers/ProfileManager.md). Если картинка не загрузится, оркестратор заменит её на заглушку.
*   **Slug**: ВАЖНО — ссылка формируется через `SlugService`. Если имя предмета изменится, ссылка обновится автоматически.
