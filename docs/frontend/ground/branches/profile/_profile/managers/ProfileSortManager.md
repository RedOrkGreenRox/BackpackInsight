# [Frontend/Web/ground/branches/profile/_profile/managers/ProfileSortManager.ts](../../../../../../../Frontend/Web/ground/branches/profile/_profile/managers/ProfileSortManager.ts)

## Назначение
Специализированный менеджер для выполнения «чистой» сортировки предметов. Он инкапсулирует математическую логику сравнения, не касаясь DOM или системных хранилищ.

## Связи (Dependencies)
*   `[profile-types.ts](../utils/profile-types.md)`: Использует интерфейс `Item`.
*   Вызывается из `[ProfileDataManager.ts](ProfileDataManager.md)` для обработки массивов данных.

---

## Подробное описание методов

### `sortItems(items, currentSort, weights)`
*   **Задача**: Статичная сортировка массива предметов.
*   **Параметры**:
    *   `items`: Массив объектов предметов.
    *   `currentSort`: Текущий режим ('rarity' или 'level').
    *   `weights`: Словарь весов редкостей (из `[rarity-weights.md](../utils/rarity-weights.md)`).
*   **Алгоритм (многоуровневый)**:
    1.  Если выбран режим **rarity**, сначала сравниваются числовые веса редкостей.
    2.  Если редкости равны (или выбран режим **level**): сравниваются уровни предметов.
    3.  Если уровни равны: вычисляется процент прогресса до следующего уровня.
    4.  Если прогресс совпадает: применяется алфавитная сортировка по имени.
*   **Immutability**: Метод создает копию массива через `[...items]`, что предотвращает нежелательные побочные эффекты в исходных данных профиля.

## AI-контекст
*   **Чистая логика**: Этот файл — идеальный пример "чистой функции". Его легко тестировать в изоляции.
*   **Производительность**: Сортировка выполняется на клиенте. Для профилей с 500+ предметами стоит учитывать время выполнения этого метода (хотя на текущих объемах данных в **Backpack Brawl** это происходит мгновенно).
