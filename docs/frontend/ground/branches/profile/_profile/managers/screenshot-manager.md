# [Менеджер скриншотов (screenshot-manager.ts)](../../../../../../../Frontend/Web/ground/branches/profile/_profile/managers/screenshot-manager.ts)

## Назначение
Этот менеджер связывает [Утилиту скриншотов](../../../../../utils/screenshotUtils.md) с интерфейсом профиля. Он готовит страницу к снимку: скрывает кнопки управления и элементы навигации, чтобы итоговое изображение выглядело как чистая «карточка игрока».

## Связи (Dependencies)
*   [Утилита скриншотов](../../../../../utils/screenshotUtils.md): Выполняет техническую работу по захвату DOM.
*   [Система i18n](../../../../localization/i18n.md): Для уведомлений пользователя о процессе сохранения.

## Ключевая логика
*   **Подготовка**: Перед снимком добавляет временный класс `.is-taking-screenshot` к контейнеру, что скрывает лишние элементы через CSS.
*   **Захват**: Вызывает рендеринг области `#profileContainer`.
*   **Завершение**: Автоматически скачивает файл с именем `profile_{nickname}.png`.

## AI-контекст
Если на скриншоте появляются лишние кнопки, проверьте, добавлены ли они в список исключений в этом файле.
