# [Frontend/Web/ground/branches/profile/_profile/managers/ProfileManager.ts](../../../../../../Frontend/Web/ground/branches/profile/_profile/managers/ProfileManager.ts)

## Назначение
Главный оркестратор страницы профиля игрока. Этот класс управляет всеми интерактивными частями страницы: от переключения скинов героев до сортировки предметов и создания скриншотов. Он объединяет специализированные менеджеры в единый рабочий процесс.

## Связи (Dependencies)
*   **Менеджеры данных и состояния**:
    *   `[ProfileDataManager.ts](./ProfileDataManager.md)`: Подготовка данных и сортировка.
    *   `[ProfileStateManager.ts](./ProfileStateManager.md)`: Сохранение позиции скролла и фильтров.
    *   `[ProfileSkinsManager.ts](./ProfileSkinsManager.md)`: Логика косметики героев.
    *   `[screenshot-manager.ts](./screenshot-manager.md)`: Функционал сохранения карточки профиля.
*   **UI Рендереры**:
    *   `[header.ts](../header/header.md)`: Отрисовка шапки профиля.
    *   `[heroes-section.ts](../heroes/heroes-section.md)`: Отрисовка сетки героев.
    *   `[items-section.ts](../items/items-section.md)`: Отрисовка коллекции предметов.
    *   `[item-card.ts](../items/item-card.md)`: Рендеринг индивидуальных карточек предметов.
*   **Системные утилиты**:
    *   `[SortController.ts](../sort/SortController.md)`: Контроллер сортировки героев.
    *   `[ImageFormatService.ts](../../../../../utils/ImageFormatService.md)`: Обработка ошибок загрузки картинок.
    *   `[i18n.ts](../../../../localization/i18n.md)`: Локализация текстов кнопок.

## Подробное описание методов

### `init()`
Основной метод запуска.
*   Использует `requestAnimationFrame` для плавной вставки HTML-структуры.
*   Рендерит шапку, кнопку скачивания, секции героев и предметов.
*   Внедряет скрытый JSON-скрипт `#skins-data` для передачи информации о скинах в DOM.
*   Вызывает `attachAll()` для регистрации событий.
*   Восстанавливает состояние страницы (скролл, скины) через `restoreDynamicState`.

### `destroy()`
Метод очистки.
*   Сохраняет текущее состояние страницы.
*   Вызывает все функции отписки из `cleanupFns`.
*   Уничтожает дочерние менеджеры (`screenshotManager`, `sortController`).

### Методы `attach...` (Регистрация событий)
*   `attachImageErrorHandler()`: Глобальный перехват ошибок загрузки изображений внутри контейнера. При сбое подставляет заглушку из `[ImageFormatService.md](../../../../../utils/ImageFormatService.md)`.
*   `attachItemSort()`: Слушает клик по кнопке сортировки предметов. Меняет режим ('rarity' <-> 'level'), пересортировывает массив через `ProfileDataManager` и обновляет DOM грида.
*   `attachSkins()`: Находит карточки героев с кнопками смены скина. Настраивает циклическое переключение и синхронизирует выбранный скин с маленькой иконкой в шапке (`updateHeaderSkin`).
*   `attachItemLinks()`: Подготавливает ссылки на предметы, сохраняя состояние страницы перед переходом на детальный просмотр.

### Методы управления состоянием (State)
*   `saveCurrentState()`: Собирает объект `SavedState`, включая текущую позицию скролла, выбранные режимы сортировки и ID надетых скинов для каждого героя. Сохраняет через `ProfileStateManager`.
*   `restoreDynamicState()`: Выполняет обратную операцию — прокручивает страницу, применяет сортировки и восстанавливает внешний вид героев.

## AI-контекст
*   **Архитектурный узел**: Если на странице профиля ломается интерактив — это первый файл для проверки. Он является клеем между версткой и бизнес-логикой.
*   **Синхронизация скинов**: Обратите внимание на метод `updateHeaderSkin`. Он обеспечивает реактивность: когда вы меняете скин на большой карточке героя, он мгновенно меняется и в "статах" игрока наверху страницы.
