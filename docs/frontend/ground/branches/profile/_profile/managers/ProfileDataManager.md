# [Frontend/Web/ground/branches/profile/_profile/managers/ProfileDataManager.ts](../../../../../../../Frontend/Web/ground/branches/profile/_profile/managers/ProfileDataManager.ts)

## Назначение
Этот менеджер отвечает за подготовку и жизненный цикл данных на странице профиля. Он решает, откуда взять данные (из аргументов навигации или кеша), как их отсортировать для пользователя и как подготовить SEO-информацию.

## Связи (Dependencies)
*   `[i18n.ts](../../../../localization/i18n.md)`: Для генерации локализованных мета-тегов и текстов в скелетонах.
*   `[LoadingStates.ts](../../../../../utils/LoadingStates.md)`: Использует методы создания скелетонов (скелетных загрузочных экранов).
*   `[ProfileStateManager.ts](./ProfileStateManager.md)`: Синхронизирует состояние (сортировки, фильтры) с данными.
*   `[profile-types.ts](../utils/profile-types.md)`: Использует интерфейсы `ProfileData` и `Item` для типизации.
*   `[rarity-weights.ts](../utils/rarity-weights.md)`: Получает веса редкостей для алгоритма сортировки предметов.

## Подробное описание методов

### `resolve(data)`
*   **Задача**: Определить активный профиль для отрисовки.
*   **Логика**: Сначала проверяет входящий объект `data`. Если он пуст (например, при прямом обновлении страницы браузером), пытается восстановить последний профиль из `sessionStorage`.
*   **Кеширование**: После успешного разрешения сохраняет данные в `CACHE_KEY`, делая их доступными для других компонентов.

### `sortItems(items, sortType)`
*   **Задача**: Упорядочить коллекцию предметов игрока.
*   **Алгоритм**:
    1.  Если `sortType === 'rarity'`: сначала сравнивает по весам из `[rarity-weights.md](../utils/rarity-weights.md)`.
    2.  Затем по **уровню** предмета (от большего к меньшему).
    3.  Затем по **прогрессу до следующего уровня** (отношение текущих карточек к нужным).
    4.  В конце — по алфавиту (название).
*   **Результат**: Сохраняет отсортированный список в `LIST_KEY` для поддержки последовательной навигации на детальной странице предмета.

### `renderSkeleton()`
*   **Задача**: Сгенерировать «заглушку» страницы на время ожидания данных.
*   **Логика**: Вызывает методы `LoadingStates`, формируя структуру: Шапка профиля -> Секция героев (3 карточки) -> Секция предметов (6 карточек).

### `getMeta(data)`
*   **Задача**: Генерация динамических мета-тегов `title` и `description`.
*   **Логика**: Если данные есть, формирует заголовок вида "Никнейм — Профиль | Backpack Insight". Это важно для SEO и отображения в закладках.

## AI-контекст
*   **SEO**: При добавлении новых данных в профиль (например, "Винрейт"), обновите логику `getMeta`, чтобы информация попадала в превью страницы.
*   **Хранилище**: Данные в `sessionStorage` хранятся только в течение сессии. При закрытии вкладки кеш очищается.
