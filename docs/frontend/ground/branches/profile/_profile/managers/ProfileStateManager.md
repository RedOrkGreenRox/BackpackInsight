# [Frontend/Web/ground/branches/profile/_profile/managers/ProfileStateManager.ts](../../../../../../Frontend/Web/ground/branches/profile/_profile/managers/ProfileStateManager.ts)

## Назначение
Этот менеджер отвечает за сохранение и восстановление динамического состояния страницы профиля. Он гарантирует, что если пользователь уйдет на другую страницу и вернется, его прокрутка (scrollY), выбранная сортировка и выбранные скины героев останутся прежними.

## Связи (Dependencies)
*   Используется `[ProfileDataManager.ts](./ProfileDataManager.md)` для первичной настройки данных.
*   Типизирует состояние через интерфейс `SavedState`.

---

## Описание интерфейса `SavedState`
Объект, описывающий состояние UI:
*   `scrollY`: Позиция вертикальной прокрутки страницы.
*   `itemSort`: Текущий режим сортировки предметов ('rarity' | 'level').
*   `heroSort`: Объект с настройками сортировки героев (поле `sortBy` и флаг `inverted`).
*   `currentSkins`: Словарь выбранных скинов для героев (например, `{"Ronan": "1"}`).

---

## Подробное описание методов

### `saveState(state)`
*   **Задача**: Физическое сохранение объекта состояния.
*   **Логика**: Дублирует данные в `sessionStorage` (для текущей сессии) и в `localStorage` (для сохранения между перезапусками браузера).

### `restoreState()`
*   **Задача**: Извлечение сохраненного состояния.
*   **Логика**: Пытается прочитать данные из хранилищ. Если данных нет или произошла ошибка парсинга, возвращает **объект по умолчанию** (default state).

### `clearState()`
*   Очищает оба хранилища. Обычно вызывается при загрузке совершенно нового профиля другого игрока.

## AI-контекст
*   **UX (User Experience)**: Этот файл — ключ к "бесшовному" интерфейсу. При добавлении новых интерактивных элементов на страницу профиля (например, фильтров), не забудьте добавить их в интерфейс `SavedState` и логику сохранения этого менеджера.
*   **Источники данных**: Менеджер использует агрессивное сохранение (session + local), чтобы минимизировать потерю данных при сбоях.
