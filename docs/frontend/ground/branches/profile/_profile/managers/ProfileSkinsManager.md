# [Frontend/Web/ground/branches/profile/_profile/managers/ProfileSkinsManager.ts](../../../../../../../Frontend/Web/ground/branches/profile/_profile/managers/ProfileSkinsManager.ts)

## Назначение
Этот менеджер управляет логикой отображения косметических скинов героев. Он разбирает данные о владении скинами из профиля игрока и формирует пути к соответствующим изображениям.

## Связи (Dependencies)
*   **Изображения**: Генерирует ссылки на ассеты в `/images/heroes/`.
*   Используется в компонентах отрисовки карточек героев для динамической смены образа.

---

## Подробное описание методов

### `parseSkinsData(jsonText)`
*   **Задача**: Превратить сырую строку JSON со скинами в типизированный словарь.
*   **Логика**: Приводит все имена героев к нижнему регистру для унификации поиска. Обрабатывает ошибки парсинга, возвращая пустой объект в случае сбоя.

### `getUniqueSkins(heroName, skinsMap)`
*   **Задача**: Сформировать полный список доступных скинов для конкретного героя.
*   **Логика**: Всегда добавляет базовый скин `01` (который есть у всех), объединяет его со списком из профиля и удаляет дубликаты через `new Set()`. Сортирует список по возрастанию номера.

### `getSkinImagePaths(heroName, skin)`
*   **Задача**: Сгенерировать пути к файлам изображений для разных форматов.
*   **Результат**: Возвращает объект с путями к `.webp` и `.avif` версиям.
*   **Паттерн именования**: `/images/heroes/{hero}/[webp|avif]/{hero}{skin}.[webp|avif]`. Например: `/images/heroes/ronan/webp/ronan02.webp`.

## AI-контекст
*   **Консистентность ассетов**: Менеджер опирается на строгий паттерн именования файлов в папке `static`. Если вы переименовываете изображения героев, обязательно обновите этот паттерн.
*   **Кеширование**: Скины — это визуально тяжелая часть страницы. Этот менеджер помогает быстро переключать изображения без лишних расчетов.
