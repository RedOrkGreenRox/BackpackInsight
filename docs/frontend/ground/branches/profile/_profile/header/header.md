# [Рендерер заголовка (header.ts)](../../../../../../../Frontend/Web/ground/branches/profile/_profile/header/header.ts)

## Назначение
`HeaderRenderer` — основной компонент для отрисовки верхней части страницы профиля. Он объединяет общую информацию об игроке, сводную статистику ресурсов и мини-карточки героев.

---

## Ключевая логика

### 1. Композиция
Метод `render` собирает заголовок из нескольких частей:
*   [**PlayerInfoRenderer**](player-info.md): Никнейм игрока.
*   [**StatsBarRenderer**](stats-bar.md): Основные ресурсы (Уровень, Трофеи, Валюта).
*   **Hero Mini-Cards**: Компактные индикаторы героев со статистикой лиги и уровня.
*   **Item Stats**: Сводная панель количества предметов по редкостям.

### 2. Форматирование рейтинга (`formatRating`)
Специальная логика для отображения прогресса внутри тира:
*   Если рейтинг < 5000, показывает остаток от деления на 500 (прогресс текущей лиги).
*   Если рейтинг > 5000, вычитает 5000 (прогресс в элитных лигах).

### 3. Индикация версий
Отображает текущую версию игры (`actual_version`) и версию, на которой был создан профиль (`install_version`), что помогает диагностировать расхождения в данных.

---

## Связи (Dependencies)
*   **Данные**: Принимает объект [`ProfileData`](../utils/profile-types.md).
*   **Изображения**: Использует динамические пути к фонам арен и иконкам лиг.

## AI-контекст
*   Компонент использует `data-aos="zoom-in"` для эффектного появления при загрузке страницы.
*   Мини-карточки героев содержат `data-` атрибуты, необходимые для работы [**SortController**](../sort/SortController.md).
