# [Рендерер заголовка профиля (HeaderRenderer)](../../../../../../../Frontend/Web/ground/branches/profile/_profile/header/header.ts)

## Назначение
Этот класс отвечает за формирование HTML-структуры верхней части страницы профиля. Он объединяет данные игрока, фон текущей арены, статистику героев и предметов в один сложный визуальный блок.

## Связи (Dependencies)
*   **Типы**: Использует [Интерфейсы профиля](../utils/profile-types.md) для корректного доступа к данным.
*   **Под-компоненты**:
    *   [Рендерер информации об игроке](player-info.md): Формирует блок ника и общего уровня.
    *   [Рендерер панели ресурсов](stats-bar.md): Отрисовывает монеты, гемы и трофеи.
*   **Изображения**: Ссылается на [фоны арен (/images/area/)](../../../../../static/images/area/index.md) и [иконки UI профиля (/images/profile/)](../../../../../static/images/profile/index.md); фон арены выбирается по полю `data.area`. Формат подбирает [ImageFormatService](../../../../utils/ImageFormatService.md).

## Подробное описание методов

### `static render(data: ProfileData)`
Главный метод сборки.
*   **Фон**: Динамически подставляет изображение арены на основе поля `data.area`. Использует [современные форматы (AVIF/WebP)](../../../../utils/ImageFormatService.md).
*   **Версия**: Отображает техническую информацию о версии игры **Backpack Brawl**.
*   **Сетка героев**: Итерирует массив героев и вызывает внутренний метод `renderHeroCard`.
*   **Статистика предметов**: Вызывает `renderItemStats` для формирования грида редкостей.

### `private static renderHeroCard(hero)`
Формирует маленькую карточку героя для горизонтального списка в заголовке.
*   **Логика рамки**: Если у героя есть `prestige`, подставляется особая «золотая» рамка уровня.
*   **Лиги**: Отображает иконку лиги героя (Bronze, Master и т.д.) на основе текстового значения из БД.
*   **Dataset**: Сохраняет метаданные (`data-hero-name`, `data-level`) прямо в HTML-узел, что позволяет [Контроллеру сортировки](../sort/SortController.md) мгновенно находить и перемещать эти карточки.

### `private static renderItemStats(data)`
Генерирует блок статистики по количеству предметов игрока.
*   **Порядок редкостей**: Использует жестко заданный массив `rarityOrder` (от Boon до Common), чтобы иконки всегда шли в одном и том же логичном порядке, независимо от того, каких предметов у игрока больше.
*   **Условная отрисовка**: Если у игрока 0 предметов определенной редкости, иконка не отрисовывается, экономя место.

### `private static formatRating(rating)`
Математический хелпер для отображения рейтинга внутри лиги.
*   **Логика**: В **Backpack Brawl** отображение очков меняется после достижения порога в 5000. Метод вычисляет остаток очков для текущего ранга, чтобы цифра на иконке лиги была корректной.

## AI-контекст
*   **Оптимизация**: Все изображения в заголовке имеют атрибут `loading="lazy"`, кроме фона арены (`fetchpriority="high"`), так как он является критическим элементом (LCP).
*   **Динамика**: Заголовок обернут в `data-aos="zoom-in"`, что активирует анимацию появления при загрузке страницы.
