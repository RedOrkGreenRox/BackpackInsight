# [Валидатор JSON (JsonValidator.ts)](../../../../../../../../Frontend/Web/ground/branches/main/_main/managers/validation/JsonValidator.ts)

## Назначение
Сложный утилитный класс для глубокой проверки синтаксиса JSON-логов. Он не только проверяет валидность данных, но и формирует подробный отчет об ошибках для пользователя.

---

## Ключевая логика

### 1. Парсинг ошибок (`parseJsonError`)
Стандартный метод `JSON.parse` выдает минимальную информацию об ошибке. `JsonValidator` расширяет это поведение:
*   Использует регулярные выражения для извлечения позиции ошибки (`position X`).
*   Вычисляет точный номер строки и колонки.
*   Распознает типы ошибок: `Unexpected end` (незавершенный JSON), `Unexpected token` (опечатка), `Property name error`.

### 2. Подсветка ошибок (`highlightError`)
Генерирует HTML-разметку «мини-редактора кода», которая:
*   Отрисовывает номера строк.
*   Выделяет конкретную строку, где произошла ошибка.
*   Маркирует красный цветом ошибочный символ.

---

## AI-контекст
*   Метод `escapeHtml` гарантирует, что если в логе содержатся вредоносные скрипты или HTML-теги, они будут отображены как обычный текст, предотвращая XSS-атаки.
*   Сервис интегрирован с системой локализации [**i18n**](../../../../../localization/i18n.md) для вывода понятных сообщений об ошибках.
