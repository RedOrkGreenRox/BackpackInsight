# [Валидатор JSON (JsonValidator.ts)](../../../../../../../Frontend/Web/ground/branches/main/_main/managers/validation/JsonValidator.ts)

## Назначение
Класс для глубокого синтаксического анализа JSON. В отличие от браузерного `JSON.parse`, он предоставляет подробную информацию о том, **где именно** и **почему** произошла ошибка, что критично для парсинга логов **Backpack Brawl**.

## Связи (Dependencies)
*   **Локализация**: Использует [Систему i18n](../../../../../localization/i18n.md) для вывода ошибок.
*   **Вызов**: Используется [Менеджером валидации](../ValidationManager.md).

---

## Подробное описание методов

### `validateJson(jsonText: string)`
*   **Задача**: Определить, является ли текст валидным JSON-объектом.
*   **Логика**: Пытается выполнить `JSON.parse`. Если успешно — возвращает `isValid: true`. Если нет — ловит исключение и передает его в `parseJsonError`.

### `private static parseJsonError(error, jsonText)`
*   **Сложность**: Реализует парсинг технических сообщений об ошибках JS-движка (V8).
*   **Логика**:
    *   Ищет паттерны типа `at position {N}` в сообщении ошибки.
    *   Если находит — вызывает `getLineAndColumn` для конвертации индекса символа в координаты.
    *   Подставляет локализованные ключи: `json_incomplete`, `json_unexpected_token`.

### `highlightError(jsonText, line, column)` — **ГЛУБОКАЯ ЛОГИКА**
*   **Задача**: Создать HTML-код редактора с подсветкой места ошибки.
*   **Алгоритм**:
    1.  Разбивает входной текст на массив строк.
    2.  Находит строку по индексу `line - 1`.
    3.  Разрезает ошибочную строку на три части: [до ошибки], [символ ошибки], [после ошибки].
    4.  Оборачивает [символ ошибки] в тег `<span class="error-char">`.
    5.  Генерирует HTML-таблицу, где слева номера строк, а справа — код.
    6.  Добавляет подпись с описанием ошибки под кодом.

### Вспомогательные методы
*   `getLineAndColumn()`: Считает количество переносов строк до указанной позиции.
*   `escapeHtml()`: Использует `textContent` временного DOM-узла для очистки строк. Это гарантирует, что если в логе игры есть символы `<` или `>`, они не сломают верстку сайта.

## AI-контекст
Этот класс — единственный источник правды о корректности входящих данных. Если пользователь вставляет битый JSON, именно этот файл отвечает за то, чтобы не "положить" всё приложение.
