# [Frontend/Web/ground/branches/main/_main/managers/DraftManager.ts](../../../../../../../Frontend/Web/ground/branches/main/_main/managers/DraftManager.ts)

## Назначение
Менеджер черновиков. Он отделяет логику управления «состоянием» черновика от низкоуровневой работы с хранилищем или событиями. Гарантирует, что данные пользователя в **Backpack Insight** не потеряются при перезагрузке страницы или случайном переходе.

## Связи (Dependencies)
*   `[StorageManager.ts](./draft/StorageManager.md)`: Для непосредственного сохранения/загрузки строк из `SessionStorage`.
*   `[DraftEventHandler.ts](./draft/DraftEventHandler.md)`: Для подписки на события ввода в `textarea`.

## Подробное описание методов

### `initDraftManagement(container)`
*   Инициализирует процесс слежения за черновиком.
*   Создает `DraftEventHandler`, передавая ему колбэк для сохранения данных через `StorageManager`.
*   **Восстановление**: Ищет сохраненный черновик в хранилище. Если он есть, находит элемент `#jsonInput` и подставляет в него текст.

### `saveDraft(data)` / `restoreDraft()` / `clearDraft()`
Публичные прослойки-хелперы, которые вызывают соответствующие методы `[StorageManager.md](./draft/StorageManager.md)`. Используются другими компонентами (например, при очистке формы после успешной отправки).

### `destroy()`
Удаляет обработчик событий `eventHandler`, очищая подписки на ввод текста.

## AI-контекст
*   **Безопасность данных**: Этот менеджер работает только с "сырым" текстом лога. Он не знает о структуре данных, его задача — просто сохранить строку.
*   **Жизненный цикл**: Всегда вызывается в `[MainManager.md](./MainManager.md)` при старте страницы.
