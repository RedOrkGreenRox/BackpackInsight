# [Frontend/Web/ground/branches/main/_main/managers/MainManager.ts](../../../../../../../Frontend/Web/ground/branches/main/_main/managers/MainManager.ts)

## Назначение
Центральный координатор (Orchestrator) главной страницы. Его роль заключается в том, чтобы связать разрозненные менеджеры (загрузки, черновиков, отправки) в единый рабочий процесс. Это позволяет разгрузить класс `[MainBranch.ts](../../MainBranch.md)` от прямой манипуляции элементами DOM и сложной бизнес-логики.

## Связи (Dependencies)
Является «мозгом» главной страницы и управляет следующими компонентами:
*   `[DraftManager.ts](./DraftManager.md)`: Управление временными данными (черновиками).
*   `[FormManager.ts](./FormManager.md)`: Работа с HTML-формой.
*   `[SubmitManager.ts](./submit/SubmitManager.md)`: Отправка данных на сервер.
*   `UploadHandler` (из `../upload-zone/upload`): Обработка событий перетаскивания и выбора файлов.

## Подробное описание методов

### `constructor(container, t)`
Принимает корневой контейнер страницы и функцию перевода `t`. Инициализирует внутренние менеджеры форм и черновиков.

### `init()`
Основной метод запуска страницы. 
*   Инициализирует `DraftManager`, восстанавливая ранее введенные данные.
*   Создает `SubmitManager` для обработки запросов.
*   Настраивает `UploadHandler`, связывая его с логикой сохранения черновиков. 
*   Связывает `FormManager` с `SubmitManager`: когда форма отправляется, вызывается асинхронная отправка данных.

### `destroy()`
Метод очистки. Вызывает `destroy()` у всех подчиненных менеджеров, обнуляет ссылки и удаляет слушатели событий, предотвращая утечки памяти при переходе на другую страницу.

### `hideError()` (private)
Вспомогательный метод для скрытия контейнера с ошибками при начале новой операции загрузки.

## AI-контекст
*   **Архитектура**: Если вы хотите изменить то, как страница реагирует на загрузку файла (например, сразу отправлять его на сервер без нажатия кнопки), изменения должны коснуться именно метода `init()` этого класса.
*   **Композиция**: `MainManager` не реализует логику сам, он только делегирует её. Если нужно изменить логику сохранения в Storage — идите в `[StorageManager.md](./draft/StorageManager.md)`.
