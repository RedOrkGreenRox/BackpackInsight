# [Оркестратор главной (MainManager.ts)](../../../../../../../Frontend/Web/ground/branches/main/_main/managers/MainManager.ts)

## Назначение
Класс `MainManager` является центральным координатором логики главной страницы. Он связывает независимые модули (загрузку, валидацию, хранение черновиков и отправку) в единый рабочий процесс.

---

## Ключевая логика

### 1. Инициализация (`init`)
При запуске менеджер настраивает взаимодействие между компонентами:
*   Активирует [**DraftManager**](DraftManager.md) для восстановления ранее введенных данных.
*   Настраивает [**UploadHandler**](../upload-zone/upload.md) для обработки перетаскиваемых файлов и вставки из буфера.
*   Инициализирует [**FormManager**](FormManager.md) для перехвата события отправки формы.

### 2. Цепочка обработки данных
Менеджер устанавливает правила:
1.  При любом изменении текста в зоне загрузки данные автоматически сохраняются в черновик.
2.  При клике на "Посмотреть профиль" вызывается [**SubmitManager**](submit/SubmitManager.md).
3.  При вводе текста старые сообщения об ошибках автоматически скрываются.

---

## Связи (Dependencies)
*   **Управляет**: `UploadHandler`, `SubmitManager`, `FormManager`, `DraftManager`.
*   **Используется**: В [**MainBranch.ts**](../../MainBranch.md) как основной исполнительный механизм.

## AI-контекст
*   Этот класс выносит всю процедурную логику из UI-класса `MainBranch`, следуя принципу единственной ответственности (SRP). Любое изменение в поведении главной страницы (например, добавление нового способа загрузки) должно начинаться с метода `init` этого класса.
