# [Frontend/Web/ground/branches/main/_main/managers/FormManager.ts](../../../../../../../Frontend/Web/ground/branches/main/_main/managers/FormManager.ts)

## Назначение
Этот класс изолирует логику работы с HTML-формой загрузки. Он отвечает за перехват события `submit` и передачу данных в колбэк-обработчик.

## Связи (Dependencies)
Напрямую взаимодействует с DOM главной страницы. Вызывается из `[MainManager.md](MainManager.md)`.

## Подробное описание методов

### `initForm(container, onSubmit)`
*   Находит элементы `#uploadForm` и `#jsonInput` внутри переданного контейнера.
*   Регистрирует асинхронный обработчик события `submit`.
*   **e.preventDefault()**: Критически важная часть — предотвращает стандартную перезагрузку страницы браузером при отправке формы.
*   Передает значение текстового поля в функцию `onSubmit`.
*   Сохраняет функцию отписки в массив `cleanupFns`.

### `destroy()`
Проходит по массиву `cleanupFns` и вызывает все сохраненные функции отписки, удаляя слушатели событий с формы. Это важно для предотвращения утечек памяти при переключении между бранчами приложения.

## AI-контекст
*   **Инкапсуляция**: Если в будущем форма станет сложнее (добавятся чекбоксы или селекторы выбора версии **Backpack Brawl**), их инициализация должна быть добавлена именно сюда.
*   **Асинхронность**: Обработчик `submit` помечен как `async`, так как отправка данных на сервер через `[SubmitManager.md](submit/SubmitManager.md)` является асинхронной операцией.
