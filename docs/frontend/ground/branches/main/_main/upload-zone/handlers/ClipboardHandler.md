# [Frontend/Web/ground/branches/main/_main/upload-zone/handlers/ClipboardHandler.ts](../../../../../../../../Frontend/Web/ground/branches/main/_main/upload-zone/handlers/ClipboardHandler.ts)

## Назначение
Реализует импорт данных профиля напрямую из буфера обмена (Ctrl+V / Paste). Один из самых удобных способов импорта в **Backpack Insight** — игроки часто просто копируют содержимое лога.

## Связи (Dependencies)
*   `[FileHandler.ts](FileHandler.md)`: импортируется напрямую (`import { FileHandler }`) — единая точка обработки контента/файлов.
*   `[JsonValidator.ts](../../managers/validation/JsonValidator.md)`: получает текст из буфера для проверки структуры.
*   `[UIHandler.ts](UIHandler.md)`: управляет визуальным состоянием зоны загрузки.
*   **Нативный DOM**: подписка на событие `paste` выполняется через `addEventListener`; результат прокидывается наверх колбэком `onContent`. Отдельной шины событий в проекте нет.

## Подробное описание (реальная реализация)

### Конструктор `(area, input, fileInput, onContent)`
*   Принимает зону загрузки, текстовое поле, скрытый `input[type=file]` и колбэк `onContent`, затем вызывает `init()`.

### Процесс обработки вставки
1.  **Перехват**: слушатель `paste` забирает `clipboardData`.
2.  **Извлечение**: пытается получить данные как `text/plain`.
3.  **Передача**: непустую строку отдаёт через `onContent`, откуда она уходит в `[JsonValidator.ts](../../managers/validation/JsonValidator.md)`.
*   Все подписки складываются в `cleanupFns` и снимаются в `destroy()`.

## AI-контекст
*   **Фокус**: следите, чтобы обработчик не перехватывал вставку, когда пользователь печатает в других потенциальных полях ввода.
*   **Безопасность**: класс работает только с текстом; изображения/бинарь игнорируются на уровне извлечения `text/plain`.
