# [Главная страница (MainBranch.ts)](../../../../../Frontend/Web/ground/branches/main/MainBranch.ts)

## Назначение
Класс `MainBranch` отвечает за отображение и логику стартовой страницы. Это основной узел приложения, через который игроки загружают свои логи для анализа.

---

## Ключевая логика

### 1. Модульный рендеринг
Страница собирается из независимых компонентов, описанных в папке `_main/`:
*   `ContainerRenderer`: Общий макет страницы.
*   `UploadZoneRenderer`: Интерактивная зона для загрузки JSON.
*   `TitleRenderer`: Заголовок и приветствие.

### 2. Управление состоянием (`MainManager`)
При инициализации бранч создает экземпляр `MainManager`. Он координирует:
*   Обработку вставленных данных из буфера обмена.
*   Валидацию JSON "на лету".
*   Переход на страницу профиля после успешной обработки.

---

## AI-контекст
*   Страница использует агрессивную локализацию (`t`), динамически подгружая тексты в зависимости от выбранного языка.
*   Метод `destroy` гарантирует очистку всех слушателей событий в зоне загрузки (Drag-and-Drop, Paste).
