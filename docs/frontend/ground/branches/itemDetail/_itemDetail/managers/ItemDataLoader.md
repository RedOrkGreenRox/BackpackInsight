# [Загрузчик данных предмета (ItemDataLoader.ts)](../../../../../../Frontend/Web/ground/branches/itemDetail/_itemDetail/managers/ItemDataLoader.ts)

## Назначение
Этот менеджер изолирует асинхронную логику получения данных для страницы [Детального просмотра предмета](../ItemDetailBranch.md). Его главная задача — обеспечить надежную загрузку через кеш или API, предоставляя интерфейс колбэков для обновления UI.

## Связи (Dependencies)
*   [Кеш предметов](../../../../../utils/ItemsCacheService.md): Первоочередной источник данных.
*   [Типы иконок предметов](../../../../../utils/ItemIconService.md): Использует интерфейс `ItemDefinition`.
*   [Оркестратор деталей предмета](./ItemDetailManager.md): Использует этот загрузчик для получения контента перед отрисовкой.

---

## Подробное описание методов

### `constructor(searchSlug: string)`
Принимает уникальный идентификатор предмета (слаг) из URL.

### Координация (Callbacks)
*   `onLoaded(cb)`: Регистрирует функцию, которая будет вызвана при успешном получении данных.
*   `onError(cb)`: Регистрирует обработчик сетевых сбоев.
*   `onNotFound(cb)`: Регистрирует действие на случай, если предмета нет в базе данных **Backpack Brawl**.

### `load(): Promise<void>` — **ГЛУБОКАЯ ЛОГИКА**
*   **Блокировка дублей**: Использует флаг `isLoading`, чтобы предотвратить одновременные повторные запросы при быстром переключении.
*   **Стратегия загрузки**:
    1.  Обращается к [Кешу предметов](../../../../../utils/ItemsCacheService.md).
    2.  Если кеш пуст, `ItemsCacheService` автоматически инициирует запрос к [Сервису API](../../../../../utils/ApiService.md).
*   **Обработка результата**: Разделяет "пустой ответ" (404) и "ошибку сервера" (500), вызывая разные цепочки уведомлений в интерфейсе.

## AI-контекст
*   **Архитектурный шаблон**: Это пример «активного загрузчика». Он не просто возвращает данные, а управляет состоянием процесса.
*   **Плавность**: Благодаря интеграции с кешем, повторные заходы на страницу предмета происходят мгновенно (Zero Latency).
