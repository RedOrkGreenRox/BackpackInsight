# [Оркестратор деталей предмета (ItemDetailManager.ts)](../../../../../../../Frontend/Web/ground/branches/itemDetail/_itemDetail/managers/ItemDetailManager.ts)

## Назначение
Центральный управляющий класс для страницы детального просмотра предмета. Он связывает загрузку данных, навигацию между предметами, SEO-оптимизацию и специфические браузерные события (например, умное копирование текста).

## Связи (Dependencies)
*   **Рендеринг**: Использует [Рендерер деталей предмета](../../../../../../../Frontend/Web/ground/branches/itemDetail/_itemDetail/components/ItemDetailRenderer.ts) для отрисовки интерфейса.
*   **Менеджеры данных**:
    *   [Загрузчик данных предмета](./ItemDataLoader.md) (Сетевые запросы).
    *   [Менеджер навигации по предметам](./ItemNavigationManager.md) (Кнопки Назад/Вперед).
    *   [SEO Менеджер](../../../../../../../Frontend/Web/ground/branches/itemDetail/_itemDetail/managers/ItemSEOManager.ts) (Заголовки и мета-теги).
*   **Утилиты**:
    *   [SlugService](../../../../../utils/SlugService.md): Для работы с URL.
    *   [Кеш предметов](../../../../../utils/ItemsCacheService.md): Для мгновенного восстановления данных.
    *   [Предзагрузка изображений](../../../../../utils/ItemPreviewPrefetchService.md): Для получения данных, если пользователь перешел по ссылке, на которую навел мышь ранее.

## Подробное описание методов

### `constructor(container, data)`
Принимает контейнер и объект данных. Вызывает `resolveData` для первичного определения, какой предмет нужно показать.

### `init()`
*   Запускает [Умное копирование](#setupcopyhandler).
*   Пытается восстановить скролл, если пользователь вернулся назад.
*   Если данные предмета уже есть в `data`, сразу вызывает `renderFullPage`.
*   Если данных нет (прямой заход по ссылке), создает [Загрузчик](./ItemDataLoader.md) и ждет ответа от API.

### `renderFullPage()` — **ГЛУБОКАЯ ЛОГИКА**
*   **Сборка страницы**: Инициализирует навигацию и SEO.
*   **Оптимизация**: Использует `requestAnimationFrame` для вставки HTML, чтобы не блокировать поток отрисовки.
*   **Связка**: Вызывает `attachProfileNavListeners`, чтобы кнопки навигации знали о состоянии предмета в профиле игрока.

### `private resolveData(data)`
*   **Задача**: "Угадать" предмет.
*   **Логика**: Если в `data` пусто, извлекает слаг из URL-адреса. Проверяет, не является ли этот заход частью просмотра профиля игрока.

### `private setupCopyHandler()` — **УНИКАЛЬНАЯ МЕХАНИКА**
*   **Задача**: Сделать копирование текста описаний удобным.
*   **Логика**: Перехватывает событие `copy`. Если пользователь выделяет текст с иконками, метод заменяет HTML-теги картинок на их текстовые эквиваленты в скобках (например, `[Poison]`).
*   **Результат**: При вставке в Discord или Telegram пользователь получит читаемый текст вместо пустых мест.

### `private restoreProfileScroll()`
Восстанавливает позицию прокрутки только при переходе "назад", используя данные из `history.state` и `sessionStorage`.

## AI-контекст
*   **Оркестрация**: Этот файл — "клей" всей страницы. Если данные загрузились, но заголовок окна не обновился — проблема в связке `renderFullPage` и `seoManager`.
*   **Память**: Метод `destroy` критически важен, так как страница предмета тяжелая и часто меняется. Он принудительно очищает ссылки на загрузчики и навигацию.
