# [Dockerfile Фронтенда (Dockerfile)](../../Frontend/Web/Dockerfile.md)

## Назначение
Инструкции для многоэтапной сборки (Multi-stage build) фронтенд-сервиса.

## Процесс сборки

### Этап 1: Builder (`node:22.12`)
1.  Установка зависимостей через `npm ci`.
2.  Компиляция проекта командой `npm run build`.
3.  Подготовка папки `dist_build` со всеми статическими ассетами.

### Этап 2: Runtime (`bun:1.1`)
1.  Использование сверхбыстрой среды Bun.
2.  Копирование скомпилированных файлов из этапа 1.
3.  Запуск сервера [server.ts](server.md).

---

## AI-контекст
*   Разделение на Builder и Runtime позволяет минимизировать финальный размер образа: в нем нет `node_modules`, исходных TS-файлов и инструментов сборки. Только Bun и готовая статика.
