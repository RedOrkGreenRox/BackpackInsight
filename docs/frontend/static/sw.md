# [Service Worker (sw.js)](../../../Frontend/Web/static/sw.js)

## Назначение
Service Worker PWA: кэширует статику, обеспечивает оффлайн-работу и стратегию обновления при деплое. Регистрируется на клиенте, перехватывает сетевые запросы.

## Связи (Dependencies)
*   Кэширует словари локализации [lang/ru.json](lang/ru.md)/[lang/en.json](lang/en.md), логотип/меню, шрифты.
*   Согласован с хеш-именами ассетов из сборки [vite.config](../vite.config.md) (хешированные файлы НЕ прекэшируются, тянутся по актуальному index.html).
*   Манифест PWA — [manifest.json](manifest.md).

## Подробное описание
*   **Версии кэшей**: `CACHE_NAME='backpack-insight-v5'`, `STATIC_CACHE`, `DYNAMIC_CACHE`; лимит `MAX_DYNAMIC_CACHE_ENTRIES=350`.
*   `STATIC_ASSETS` — стабильные unhashed ресурсы (lang, logo, menu, шрифты), прекэшируются на `install`.
*   **`install`** — открывает `STATIC_CACHE`, `addAll(STATIC_ASSETS)`, `skipWaiting()`.
*   **`activate`** — удаляет старые версии кэшей, включает `navigationPreload`, `clients.claim()`.
*   **`fetch`** — пропускает не-http и POST; для HTML-навигации — network-first (чтобы F5 после деплоя давал свежую версию); для остального — стратегии в зависимости от типа соединения (`getEffectiveConnectionType`) с ограничением размера динамического кэша.
*   **`message`** — обрабатывает сообщения только от своего origin (защита от чужих, фикс SonarCloud).

## AI-контекст
*   Это файл уровня исходного кода (PWA-логика), а не статичный ассет. При смене формата сборки/имён критичных ресурсов синхронизируйте `STATIC_ASSETS` и поднимайте версию кэша (`v5`→`v6`), иначе у пользователей останется старый кэш. HTML всегда network-first — намеренно, ради корректных обновлений.
