# [Docker Compose — продакшн (docker-compose.server.yml)](../docker-compose.server.yml)

## Назначение
Файл `docker-compose.server.yml` является специализированной версией конфигурации для развертывания проекта в **продуктовой среде** (на боевом сервере). Он сфокусирован на безопасности, стабильности и эффективном распределении ограниченных ресурсов сервера (VPS).

---

## Ключевые отличия от локальной версии

### 1. Безопасность (Security)
*   **Изоляция БД**: Порт `5432` базы данных вообще не пробрасывается на хост-машину. Взаимодействие между бэкендом и БД происходит внутри закрытой Docker-сети.
*   **Конфигурация**: Используется `env_file: - .env`, что позволяет централизованно управлять секретами (такими как `API_SECRET`) без их явного упоминания в коде compose-файла.

### 2. Управление ресурсами (Resource Limits)
Для предотвращения падения сервера из-за нехватки памяти (OOM), установлены жесткие лимиты:
*   **БД (`db`)**: Лимит **300 MB**. Этого достаточно для обслуживания PostgreSQL в рамках данного проекта и предотвращает «раздувание» процесса.
*   **Бэкенд (`backend`)**: Лимит **400 MB**.

### 3. Иммутабельность (Immutability)
*   В отличие от dev-версии, здесь **отсутствует монтирование исходного кода** через тома (`volumes`). Контейнер использует код, который был «запечен» в образ на этапе сборки. Это гарантирует, что запущенная версия в точности соответствует собранному образу.

---

## Специфика сервисов

*   **`db`**: Сохраняет данные в томе `postgres_data`. Использует healthcheck для подтверждения готовности.
*   **`backend`**: Пробрасывает порт `8000:8000` для доступа извне (или через Nginx/Cloudflare Tunnel).

---

## Связи (Dependencies)
*   **CI/CD**: Этот файл является основным для GitHub Actions пайплайна [deploy.yml](.github/workflows/deploy.md).
*   **Сборка**: Использует те же Docker-файлы, что и dev-версия, обеспечивая консистентность окружения.

## AI-контекст
*   Этот файл оптимизирован для работы на **небольших VPS** (например, с 1-2 ГБ ОЗУ).
*   При обновлении проекта на сервере рекомендуется использовать команду: 
    `docker compose -f docker-compose.server.yml up -d --build`
