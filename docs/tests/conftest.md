# [Инфраструктура тестирования (conftest.py)](../../tests/conftest.py)

## Назначение
Файл `conftest.py` является глобальной точкой конфигурации для **pytest**. Он описывает фикстуры (fixtures) — переиспользуемые объекты и состояния, которые автоматически подставляются в тесты.

---

## Ключевые фикстуры

### 1. `engine`
Создает временную базу данных SQLite **в оперативной памяти** (`sqlite://`) для каждого теста. Это гарантирует полную изоляцию тестов и высокую скорость их выполнения.

### 2. `session`
Готовит сессию БД для теста:
*   Очищает кэш [ProfileFactory](../backend/playerdata/services/ProfileFactory.md).
*   Предзагружает статические определения предметов из JSON в in-memory БД.
*   Обеспечивает актуальное состояние базы перед каждым тестом.

### 3. `client`
Предоставляет `TestClient` из FastAPI. 
*   **Важно**: Фикстура подменяет (через `dependency_overrides`) реальную базу данных на тестовую in-memory сессию. Тесты API не затрагивают физическую БД на диске.

### 4. `profile_json_minimal` / `full`
Загружает синтетические примеры JSON-логов из папки `fixtures/` для тестирования парсинга.

---

## AI-контекст
*   Скрипт автоматически устанавливает `USE_SQLITE=True` в окружении, предотвращая случайное удаление данных из PostgreSQL при запуске тестов.
*   Все тесты бэкенда зависят от этого файла. Если нужно изменить логику инициализации данных, правки вносятся именно здесь.
