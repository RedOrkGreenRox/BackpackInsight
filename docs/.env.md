# [Пример переменных окружения (.env.example)](../.env.example)

## Назначение
Файл `.env.example` служит официальным шаблоном (blueprint) для создания локального файла `.env`. Он содержит список всех необходимых переменных окружения, которые требуются для работы бэкенда, базы данных и инструментов развертывания.

---

## Ключевые параметры

### 1. Настройка базы данных (PostgreSQL)
*   **`POSTGRES_USER` / `POSTGRES_PASSWORD`**: Логин и пароль для администратора БД.
*   **`POSTGRES_DB`**: Название создаваемой базы данных.
*   **`POSTGRES_PORT`**: Порт, на котором будет доступна БД (по умолчанию `5432`).

### 2. Режимы работы
*   **`USE_SQLITE`**: 
    *   `False` (по умолчанию) — использовать PostgreSQL (рекомендуется для Docker).
    *   `True` — переключить проект на использование SQLite. Полезно для быстрой отладки кода без необходимости поднимать контейнеры с БД.

---

## Прочие важные переменные (для Production)
Хотя в шаблоне указаны базовые параметры, в реальном `.env` на сервере также настраиваются:
*   **`API_SECRET`**: Секретный ключ для защиты API (см. [api.py](backend/playerdata/api.md)).
*   **`CORS_ORIGIN`**: Список разрешенных доменов для фронтенда.
*   **`PROFILE_RETENTION_DAYS`**: Срок хранения профилей игроков.

---

## AI-контекст
*   **Безопасность**: Файл `.env` добавлен в `.gitignore` и **никогда** не должен попадать в публичный репозиторий.
*   При первом развертывании выполните команду: `cp .env.example .env` и измените стандартные значения (admin/secret) на свои.
