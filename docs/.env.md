# [Пример переменных окружения (.env.example)](../.env.example)

## Назначение
Шаблон `.env`: учётные данные БД и настройки приложения. Копируется в `.env` перед запуском Docker.

## Переменные
*   `POSTGRES_USER/PASSWORD/DB`, `POSTGRES_PORT`.
*   `USE_SQLITE` — `True` для локальной разработки без Docker (SQLite вместо PostgreSQL).
*   (В проде также `API_SECRET`, `CORS_ORIGIN`, `RATE_LIMIT_PROFILES`, `PROFILE_RETENTION_DAYS` — см. [api.py](backend/playerdata/api.md), [database.py](backend/db/database.md).)

## AI-контекст
*   Реальный `.env` в репозиторий не коммитится. CD-пайплайн ([deploy.yml](.github/workflows/deploy.md)) конвертирует `.env` в Unix-формат на сервере.
