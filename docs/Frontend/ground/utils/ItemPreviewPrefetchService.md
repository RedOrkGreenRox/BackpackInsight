# [Предзагрузка предметов (ItemPreviewPrefetchService.ts)](../../../../Frontend/Web/ground/utils/ItemPreviewPrefetchService.ts)

## Назначение
Сервис для мгновенного перехода между страницами. Он предугадывает действия пользователя и загружает данные и иконки предметов **до того**, как на них кликнули.

---

## Ключевая логика

### 1. Prefetch при наведении
Когда пользователь наводит курсор на карточку в библиотеке или профиле:
*   Данные предмета (`itemData`) сохраняются в оперативную память.
*   Браузеру дается команда начать скрытую загрузку соответствующего изображения.

### 2. Мгновенная отрисовка
При клике на предмет [**ItemDetailManager**](../../ground/branches/itemDetail/_itemDetail/managers/ItemDetailManager.md) сначала проверяет этот сервис. Если данные уже загружены, страница отрисовывается без обращения к API, что создает эффект "нулевой задержки".

---

## AI-контекст
*   Использование `Map` для хранения превью позволяет быстро находить данные по слагам.
*   Метод `preloadImage` использует `img.decoding = 'async'`, чтобы процесс загрузки иконки не блокировал анимации интерфейса.
