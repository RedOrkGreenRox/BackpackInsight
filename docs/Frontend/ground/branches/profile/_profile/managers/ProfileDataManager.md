# [Менеджер данных профиля (ProfileDataManager.ts)](../../../../../../../Frontend/Web/ground/branches/profile/_profile/managers/ProfileDataManager.ts)

## Назначение
`ProfileDataManager` — это сервисный слой для работы с данными игрока на клиенте. Он отвечает за извлечение данных из кэша, их подготовку (сортировку) и генерацию предварительных состояний (Skeletons).

---

## Ключевая логика

### 1. Разрешение данных (`resolve`)
Метод пытается получить данные профиля из двух источников:
1.  **Аргумент навигации**: Если пользователь пришел с главной страницы после отправки формы.
2.  **SessionStorage**: Если пользователь обновил страницу (F5).
После извлечения данные сохраняются обратно в кэш для обеспечения персистентности.

### 2. Алгоритм сортировки инвентаря (`sortItems`)
Реализует многоуровневую логику упорядочивания предметов:
*   **Приоритет**: По весу редкости (через [**rarity-weights.ts**](../utils/rarity-weights.md)) или уровню.
*   **Вторично**: По проценту прогресса до следующего уровня (текущие карты / требуемые карты).
*   **Третично**: Алфавитный порядок.
Результат сортировки кэшируется в `profileItemsList` для обеспечения корректной навигации «Вперед/Назад» на странице деталей предмета.

### 3. Skeleton Screen (`renderSkeleton`)
Генерирует HTML-заглушки (серые анимированные блоки) через [**LoadingStates**](../../../../utils/LoadingStates.md). Это обеспечивает мгновенный визуальный отклик интерфейса, пока данные профиля «разворачиваются» в памяти.

---

## Связи (Dependencies)
*   **Типы**: Использует интерфейсы из [`profile-types.ts`](../utils/profile-types.md).
*   **Сортировка**: Зависит от словаря весов [`rarity-weights.ts`](../utils/rarity-weights.md).
*   **Состояние**: Взаимодействует с [`ProfileStateManager.ts`](ProfileStateManager.md).

## AI-контекст
*   Метод `getMeta` обеспечивает динамическое SEO: если зайти на страницу профиля по прямой ссылке, в заголовке вкладки всё равно будет никнейм игрока (если он есть в кэше).
