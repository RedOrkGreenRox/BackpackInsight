# [Оркестратор деталей (ItemDetailManager.ts)](../../../../../../../Frontend/Web/ground/branches/itemDetail/_itemDetail/managers/ItemDetailManager.ts)

## Назначение
Главный менеджер страницы предмета. Он координирует процессы загрузки данных, восстановления состояния профиля, управления SEO и навигации.

---

## Ключевая логика

### 1. Разрешение данных (`resolveData`)
Менеджер умеет восстанавливать контекст страницы из разных источников:
*   **Прямой заход**: Парсит название предмета из URL и загружает его через [`ItemDataLoader`](ItemDataLoader.md).
*   **Из профиля**: Пытается найти данные о прогрессе игрока (уровень, карты) в `sessionStorage`, чтобы отобразить персонализированную статистику.
*   **Кэш превью**: Использует [`ItemPreviewPrefetchService`](../../../../utils/ItemPreviewPrefetchService.md) для мгновенного отображения данных, если пользователь только что навел на карточку в Вики.

### 2. Умное копирование (`setupCopyHandler`)
Скрипт перехватывает событие `copy` и модифицирует содержимое буфера обмена:
*   Заменяет графические иконки (Damage, Burn) на их текстовые аналоги в квадратных скобках (напр. `[Poison]`).
*   Это позволяет пользователям копировать описания предметов в Discord или Telegram с сохранением смысла спецсимволов.

---

## AI-контекст
*   Менеджер восстанавливает позицию скролла страницы профиля (`restoreProfileScroll`) при нажатии кнопки "Назад", обеспечивая бесшовный UX.
*   Все побочные эффекты (SEO-теги, JSON-LD) очищаются в методе `destroy()`.
