# [Буфер обмена (ClipboardHandler.ts)](../../../../../../../../Frontend/Web/ground/branches/main/_main/upload-zone/handlers/ClipboardHandler.ts)

## Назначение
Менеджер взаимодействия с буфером обмена. Отвечает за вставку JSON-данных из памяти устройства и автоматическое открытие диалога выбора файлов.

---

## Ключевая логика

### 1. Обработка клика
Скрипт синхронно открывает системное окно выбора файла при клике на любую часть зоны загрузки. 
*   **Нюанс**: Если клик произошел прямо по текстовому полю (`textarea`), в котором уже есть текст, открытие файла блокируется, чтобы пользователь мог редактировать вставленные данные.

### 2. Перехват события вставки (`paste`)
Реализована поддержка двух сценариев:
1.  **Вставка текста**: Если в буфере находится строка (JSON), браузер сам вставляет её в `textarea`.
2.  **Вставка файла**: Если пользователь скопировал файл лога и нажал Ctrl+V, обработчик извлекает его через `clipboardData.items`, предотвращает стандартное поведение браузера и отправляет файл в [FileHandler.ts](FileHandler.md).

---

## Связи (Dependencies)
*   **Использует**: [FileHandler.ts](FileHandler.md) для чтения бинарных данных.
*   **Координирует**: Вызывает колбэк `onContent`, который запускает цепочку валидации в `MainManager`.

## AI-контекст
*   Метод `init` использует массив `cleanupFns` для хранения функций отписки. Это критично для предотвращения утечек памяти при смене страниц (бранчей), так как слушатели навешиваются на элементы, которые могут быть удалены из DOM.
