# [Runtime-сервер фронтенда (server.ts)](../../Frontend/Web/server.ts)

## Назначение
`server.ts` — это легковесный сервер на базе **Bun**, который обслуживает фронтенд внутри Docker-контейнера. Он отвечает за раздачу статики, проксирование API и работу SPA-навигации.

---

## Ключевая логика

### 1. Проксирование API (`/api`)
Все запросы, начинающиеся с `/api`, автоматически перенаправляются на бэкенд-сервис. Это позволяет фронтенду общаться с API без настройки CORS в браузере (через внутреннюю сеть Docker).

### 2. Раздача статики
Сервер ищет файлы в двух местах:
1.  **`/static`**: Исходные картинки, шрифты и локализация.
2.  **`/dist_build`**: Скомпилированные Vite ассеты (JS/CSS чанки).

### 3. SPA Fallback
Для корректной работы маршрутизации (когда пользователь заходит сразу на `/items` или `/profile`), сервер отдает файл `index.html` для всех путей, которые не являются прямыми запросами к файлам (картинкам/скриптам).

### 4. Управление Кэшированием
Сервер автоматически выставляет HTTP-заголовки:
*   **`CACHE_LONG`**: Для шрифтов и собранных ассетов (1 час).
*   **`CACHE_SHORT`**: Для динамических ресурсов (1 минута).
*   **`no-cache`**: Для главного HTML-файла, чтобы пользователь всегда получал актуальную версию приложения.

---

## AI-контекст
*   Сервер спроектирован для работы с **уже собранным** проектом. Он не запускает Vite в рантайме, что делает его крайне быстрым и экономным по памяти.
*   Использование Bun позволяет обрабатывать тысячи запросов в секунду на минимальных серверных мощностях.
