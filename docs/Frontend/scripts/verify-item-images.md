# [Проверка изображений предметов (verify-item-images.js)](../../../Frontend/Web/scripts/verify-item-images.js)

## Назначение
Скрипт `verify-item-images.js` — это инструмент контроля качества ассетов. Он проверяет наличие необходимых иконок предметов для всех записей в базе данных, предотвращая появление "битых" картинок в интерфейсе.

---

## Ключевая логика

### 1. Маппинг и Смаскированные предметы
Скрипт учитывает, что некоторые игровые предметы используют графику других объектов. Это описано в словаре `MASKED_ITEMS` (например, `Feral Cat` использует иконку `black-cat`).

### 2. Поддержка Heist Plans (Special)
Для предметов редкости `Special` реализована логика распознавания римских цифр в тултипах (например, `Step I`, `Step II`), что позволяет сопоставлять их с соответствующими иконками планов ограбления.

### 3. Проверка форматов
Для каждого предмета скрипт проверяет наличие файлов в двух обязательных форматах:
*   **WebP**: Основной формат.
*   **AVIF**: Формат с максимальным сжатием.

---

## Применение (Usage)
Запускается через `npm run verify:images`. Если хотя бы одна иконка отсутствует, процесс завершается с ошибкой (`exit 1`), что блокирует некорректный деплой.

## AI-контекст
*   Скрипт автоматически находит последнюю версию `items_*.json` в папке бэкенда, гарантируя актуальность проверки.
*   Функция `toSlug` используется для генерации имен файлов из названий предметов, что должно совпадать с логикой [SlugService.ts](../../ground/utils/SlugService.md).
