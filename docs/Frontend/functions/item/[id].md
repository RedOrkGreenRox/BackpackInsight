# [SSR Карточки предмета ([id].ts)](../../../../Frontend/Web/functions/item/[id].ts)

## Назначение
Cloudflare Pages Function для реализации **Server Side Rendering (SSR)** страниц предметов. Она обеспечивает "красивые" превью при репосте ссылок на конкретные предметы в социальные сети.

---

## Ключевая логика

### 1. Подмена Мета-тегов
Функция скачивает базовый `index.html` и динамически заменяет в нем стандартные мета-теги на специфичные для предмета:
*   **Title**: Название предмета.
*   **OG:Image**: Прямая ссылка на иконку предмета.
*   **Description**: Краткое описание из тултипов.

### 2. Внедрение JSON-LD
Автоматически генерирует и вставляет в `<head>` структурированные данные `Schema.org`, что позволяет поисковикам отображать расширенные сниппеты (Rich Snippets) с ценой и редкостью предмета в выдаче.

### 3. Обработка URL
Использует систему слагов, совместимую с [**SlugService**](../../ground/utils/SlugService.md), для поиска предмета в базе по его идентификатору из URL.

---

## AI-контекст
*   Эта функция работает **до** загрузки основного JS-бандла клиента. Благодаря этому поисковые боты видят контент страницы, даже если они не умеют исполнять JavaScript.
*   Функция использует `fetch` к внутреннему API `/api/items` для получения актуальных данных о характеристиках.
