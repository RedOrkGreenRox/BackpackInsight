# [Прокси-функция API ([[path]].ts)](../../../../Frontend/Web/functions/api/[[path]].ts)

## Назначение
Эта Cloudflare Pages Function работает как «умный» прокси-сервер. Она перехватывает все запросы к `/api/*` на фронтенде и перенаправляет их на физический VPS-сервер бэкенда.

---

## Ключевая логика

### 1. Безопасность и Секреты
Функция автоматически внедряет заголовок `X-Internal-Secret` во все исходящие запросы к бэкенду. Это позволяет держать секретный ключ API в защищенном хранилище Cloudflare, не раскрывая его в коде фронтенда.

### 2. Edge-кэширование
Реализована стратегия кэширования на уровне узлов Cloudflare (Edge):
*   **Таргет**: `GET /api/items`.
*   **Механизм**: Ответы бэкенда сохраняются в `caches.default` с заголовком `s-maxage=60`. Это означает, что в течение минуты тысячи пользователей могут получать список предметов, не нагружая реальный бэкенд.

### 3. Обработка сбоев
В случае недоступности бэкенда (например, при перезагрузке контейнера), функция возвращает структурированную ошибку `502 VPS_OFFLINE`.

## AI-контекст
*   Использование `[[path]]` в названии файла позволяет функции обрабатывать любые уровни вложенности путей API.
*   Функция очищает заголовок `if-range`, чтобы предотвратить конфликты при частичной загрузке данных через кэш Cloudflare.
