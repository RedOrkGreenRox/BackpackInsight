# [Заголовки безопасности и Кэширования (_headers)](../../../Frontend/Web/_headers)

## Назначение
Файл `_headers` настраивает поведение HTTP-ответов сервера (Cloudflare Pages / Netlify). Он управляет тем, как браузеры должны кэшировать файлы и защищать соединение.

---

## Стратегии Кэширования

### 1. Долгосрочное (1 год, `immutable`)
Для файлов, которые никогда не меняются (или меняют имя при изменении):
*   **JS/CSS Чанки**: `/assets/*.js`, `/assets/*.css`.
*   **Шрифты**: `/assets/fonts/*.woff2`.

### 2. Среднесрочное (1 месяц)
Для оптимизированных изображений:
*   **Картинки**: `/assets/images/*.webp`, `*.avif`.

### 3. Отсутствие кэша (`no-cache`)
Для файлов, требующих мгновенного обновления:
*   **HTML**: `/index.html`. Браузер обязан проверить актуальность версии перед использованием.
*   **API**: `/api/*`. Ответы бэкенда никогда не кэшируются на уровне заголовков сервера.

---

## AI-контекст
*   Этот файл критически важен для производительности. Благодаря `immutable` заголовкам, при повторном визите сайт загружается за миллисекунды, так как 90% ресурсов берутся из локального диска пользователя без запроса к серверу.
