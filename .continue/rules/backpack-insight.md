# Инструкции для работы с BackpackInsight

## Общие правила
- Язык общения: Русский.
- Стиль: Прямой, технический, без лишней вежливости.
- Соблюдай существующую архитектуру проекта (Dendritic architecture).
- **Важно:** НИКОГДА не создавай код, если об этом не попросили явно (см. Saved Information).

## Контекст репозитория (Приоритетные файлы)
При анализе архитектуры или внесении правок всегда опирайся на эти файлы:
1. **docker-compose.yml** (корень) — логика взаимодействия сервисов, томов и проброса портов.
2. **Frontend/Web/Dockerfile** — двухэтапная сборка (Bun + Python) и изоляция `node_modules`.
3. **Frontend/Web/web.py** — основной сервер фронтенда, логика роутинга Jinja2 и монтирования статики.
4. **Frontend/Web/vite.config.js** — конфигурация сборки ассетов, алиасы и правила именования файлов в `dist`.
5. **Backend/PlayerData/main.py** (или основной файл API) — логика бэкенда.
6. **structure.txt** — актуальная карта всех файлов проекта.

## Технологический стек
- **Backend:** Python 3.13, FastAPI, PostgreSQL.
- **Frontend Core:** Python (FastAPI + Jinja2 Templates).
- **Frontend Assets:** TypeScript, SCSS, Bun, Vite.
- **Инфраструктура:** Docker (изолированные тома для `dist` и `node_modules`).

## Специфические правила кодинга
### Python
- Следование стандартам PEP 8.
- Обязательное использование Type Hinting.
- Для работы с путями использовать только `pathlib`.

### Frontend (Vite + SCSS)
- Проект использует "Clean Stack" (HTML/SCSS/TS).
- Стили разбиты на `roots` (базовые) и `branches` (компоненты страниц).
- **Пути:** Учитывай, что Vite собирает файлы в `/app/dist_build`, который находится ВНЕ монтируемой папки проекта (для чистоты хоста).
- При правке шаблонов помнить, что изображения берутся из `/static`, а скомпилированные стили — из `/dist`.

### Docker & Environment
- Не предлагай правки, которые заставляют Docker создавать `node_modules` или `dist` на хосте (Windows). Используй изоляцию путей, как в текущем Dockerfile.
