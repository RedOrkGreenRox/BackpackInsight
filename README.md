# BackpackInsight

**BackpackInsight** — это комплексная аналитическая платформа для игры **Backpack Brawl**. Проект предназначен для глубокого анализа игровых профилей, предметов, механик боя и построения эффективных стратегий (билдов).

## 🧠 Концепция сетевой документации

Проект использует уникальную систему **атомарной сетевой документации**. Документация здесь — это не просто вспомогательный файл, а **живой цифровой двойник** кодовой базы, максимально оптимизированный для понимания ИИ и человеком.

### Принципы работы с кодом и документацией (Закон проекта):
1.  **Документация ПРЕЖДЕ кода**: При любых изменениях вы обязаны **СНАЧАЛА** внести изменения в соответствующие `.md` файлы документации (или создать новые/удалить старые). Документация — это ваше ТЗ и архитектурный план.
2.  **Реализация**: Только после того, как "сеть" документации обновлена, вы приступаете к изменению кодовой базы (`.ts`, `.py`, и т.д.).
3.  **Финальная истина**: После завершения кодинга документация должна быть отредактирована еще раз, чтобы устранить любые расхождения, возникшие в процессе реализации. В сети должна остаться только **абсолютная истина**.
4.  **Принцип 1:1**: Каждый файл исходного кода имеет свой зеркальный `.md` файл.
5.  **Ссылочная целостность**: Вместо повторного описания логики используйте ссылки на документацию зависимых файлов. Ссылки должны быть оформлены как человекочитаемый текст: `[Название компонента](путь_к_документации.md)`.

## 🛠 План реструктуризации

1.  📦 **Распил монолита «Предметы»**: Декомпозиция логики предметов на независимые модули.
2.  🧼 **«Гигиена»**: Глобальная чистка кода, типизация и унификация стилей.
3.  🧱 **Пересмотр чанков**: Оптимизация разбиения чанков Vite для эффективной загрузки.
4.  🧪 **Единая сеть тестов**: Создание сквозного тестирования (Frontend <-> Link <-> Backend).
5.  📚 **Сетевая документация**: Наполнение и связывание всех узлов графа (Текущий этап).
